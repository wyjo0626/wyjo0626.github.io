Atom: Low-Bit Quantization for Efficient and Accurate LLM Serving

논문 및 이미지 출처 : https://arxiv.org/pdf/2310.19102

Abstract

content generation, intelligent chatbots, sentiment analysis 와 같은 application 에서 Large Language Models (LLMs) 에 대한 수요가 증가함에 따라, LLM service provider 에게는 상당한 도전 과제가 제기된다. GPU resource 를 효율적으로 사용하고 throughput 을 향상시키기 위해, 여러 request 를 batching 하는 방식이 널리 사용되는 paradigm 으로 자리 잡았다. 또한 batching 을 더욱 가속하기 위해, LLM quantization 기법은 memory consumption 을 줄이고 computing capacity 를 증가시킨다.

그러나 기존의 quantization scheme (e.g., 8-bit weight-activation quantization) 은 4-bit integer operator 와 같은 modern GPU 의 capability 를 충분히 활용하지 못하며, 그 결과 sub-optimal performance 를 초래한다.

LLM 의 serving throughput 을 극대화하기 위해, 저자는 미미한 accuracy loss 만으로 높은 throughput 향상을 달성하는 low-bit quantization 방법인 Atom 을 제안한다.

  • Atom 은 low-bit operator 를 사용하여 serving throughput 을 크게 향상시키고, low-bit quantization 을 통해 memory consumption 을 상당히 감소시킨다.
  • 또한 새로운 mixed-precision 및 fine-grained quantization process 를 적용함으로써 높은 accuracy 를 달성한다.

저자는 serving context 에서 4-bit weight-activation quantization 에 대해 Atom 을 평가한다. Atom 은 동일한 latency target 을 유지하면서, FP16 대비 최대 7.73 ×, INT8 quantization 대비 2.53 × 의 end-to-end throughput (token/s) 향상을 달성한다.

1 Introduction

Large Language Models (LLMs) 은 요약, code completion, decision-making 과 같은 작업에 활용되며, 점점 더 업무 환경과 일상생활에 통합되고 있다. 연구에 따르면 ChatGPT 는 1 억 명 이상의 사용자를 보유하고 있으며, 매월 10 억 회 이상의 웹사이트 접속이 발생한다. 또한 LLM 의 크기와 capability 는 더 다양한 task 를 수용하기 위해 지속적으로 증가하고 있다. 높은 inference 수요와 model complexity 로 인해, LLM service provider 의 운영 비용, 즉 compute/memory 및 energy 비용은 하루 약 1 million1 \text{ million}1 million 에 근접할 정도로 크게 증가하였다.

이러한 배경에서 LLM serving 을 최적화하는 문제는 시급한 과제가 되었다. 대부분의 연구는 다양한 사용자로부터의 request 를 batching 하여 LLM serving throughput 을 향상시키는 데 초점을 맞추고 있다. 여러 request 를 batching 하면 compute intensity 가 증가하고 weight matrix 를 load 하는 비용이 amortize 되어 throughput 이 향상된다. 기존 연구는 batching 효율을 더욱 개선하기 위해 LLM quantization 기법을 탐구하였다. 이러한 기법은 16-bit floating point (FP16) 값을 더 작은 data type 으로 대체하여 memory consumption 을 줄이고 computation 을 가속한다.

그러나 현재의 quantization scheme 은 emerging low-bit hardware support (e.g., Nvidia Ampere 및 Qualcomm Hexagon) 가 제공하는 capability 를 충분히 활용하지 못한다. 예를 들어, 일부 기존 접근은 weight-only quantization 을 탐구하였다. 이러한 scheme 에서는 weight 를 low-bit representation (e.g., INT3) 으로 quantize 하지만, activation 은 floating point representation (e.g., FP16) 으로 유지된다. 따라서 activation 과 floating point 연산을 수행하기 전에 weight 를 다시 적절한 floating point representation (e.g., FP16) 으로 dequantize 해야 한다. 그 결과 weight-only quantization 은 memory consumption 을 줄이지만, 여전히 cost 가 큰 floating-point arithmetic 을 요구하며, 특히 large batch size 에서 비효율적이다.

또 다른 주요 quantization scheme 은 weight-activation quantization 이다. 이 방식에서는 weight 와 activation 을 모두 low-bit representation 으로 quantize 한다. 이 경우 weight 와 activation 을 low-precision arithmetic unit 을 사용하여 직접 곱할 수 있다. 이러한 quantization 접근은 효율적인 low-bit hardware support 덕분에 weight-only quantization 보다 더 높은 inference throughput 을 달성할 잠재력이 있다. 예를 들어, A100 GPU 는 Tensor Core 기준으로 FP16 에서 312 TFLOPS 를 제공하는 반면, INT4 에서 1248 TOPS, INT8 에서 624 TOPS 를 달성할 수 있다. LLM.INT8() 및 SmoothQuant 와 같은 기존 연구는 INT8 weight-activation quantization 을 탐구하여 거의 accuracy loss 없이 성능을 달성하였다.

그러나 INT8 quantization 역시 INT4 Tensor Core 와 같은 더 낮은 bit arithmetic 을 활용하지 못한다. 또한 INT8 quantization 은 LLM serving 에서의 대규모 memory consumption 을 줄이는 데 여전히 sub-optimal 하다. 이때 model parameter 와 batching 된 KV-cache 가 모두 큰 memory 를 소비한다.

lower-bit 의 weight-activation quantization 에 대해, OmniQuant 및 QLLM 과 같은 최근 연구는 LLM 을 4-bit 까지 quantize 하는 방법을 제안하였다. 그러나 Fig. 2 에서 보이듯이, 이들 기법은 FP16 baseline 대비 여전히 상당한 perplexity 증가를 보인다. 따라서 hardware efficiency 를 유지하면서 LLM 을 low-bit representation 으로 정확하게 quantize 하는 방법은 여전히 open research 문제이다.

본 연구에서 저자는 modern hardware 를 효율적으로 활용하는 정확한 low-bit weight-activation quantization 방법인 Atom 을 제안한다. accuracy 를 유지하기 위해 Atom 은 세 가지 핵심 quantization 설계를 포함한다.

  • (1) mixed-precision quantization 을 채택하여, 일부 중요 activation 과 weight 를 high precision 으로 유지함으로써 accuracy 를 보존한다.
  • (2) weight 와 activation 모두에 대해 fine-grained group quantization 을 적용하여 quantization error 를 자연스럽게 감소시킨다.
  • (3) activation 에 대해 quantization parameter 를 사전에 계산하지 않고, 각 input 의 distribution 을 가장 잘 반영하도록 activation 을 dynamic quantization 한다.

이러한 최적화는 quantization accuracy 를 향상시킬 수 있지만, bespoke design 없이는 hardware 를 효율적으로 활용하지 못할 수 있다. 예를 들어, mixed-precision 기법은 irregular memory access 와 성능 저하를 초래할 수 있으며, group quantization 을 사용하는 matrix multiplication 은 kernel library 에서 잘 지원되지 않는다. 또한 activation 의 dynamic quantization 은 추가 computation overhead 를 유발한다.

높은 hardware efficiency 를 보장하고 quantization overhead 를 최소화하기 위해 Atom 은 다음과 같은 설계를 적용한다.

  • (1) mixed-precision 연산에서 regular memory access 를 유지하기 위해 activation 과 weight 를 재정렬한다.
  • (2) quantization 및 재정렬 연산을 기존 operator 에 fuse 하여 overhead 를 완화한다.
  • (3) outlier 를 8-bit 로 추가 quantize 하여 accuracy 와 efficiency 사이의 균형을 유지한다.
  • (4) KV-cache 를 low-bit representation 으로 quantize 하여 memory movement 를 줄인다.

Atom 의 quantization workflow 는 Fig. 1 에 제시된다.

Atom 의 실현 가능성을 검증하기 위해, 저자는 이를 end-to-end serving framework 에 통합하였다.

  • mixed-precision 및 group quantization 을 포함하는 특수한 matrix multiplication 을 위해, low-bit tensor core 를 활용하는 customized CUDA kernel 을 구현하였다.
  • popular dataset 에 대한 실험 결과, weight 와 activation 을 모두 4-bit 로 quantize 하였을 때 Atom 은 거의 무시할 수 있는 accuracy loss (Llama-65B 기준 평균 zero-shot accuracy 1.4% 감소, WikiText2 perplexity 0.3 증가) 를 보였다.
  • 반면 동일 precision 에서 기존 방법은 더 큰 accuracy loss 를 보였다 (Tab. 1 참조).

end-to-end serving throughput 을 다양한 precision 및 quantization scheme 과 비교한 결과, Atom 은 유사한 latency 를 유지하면서 FP16 대비 최대 7.7 ×, W4A16 대비 5.5 ×, W8A8 대비 2.5 × 의 throughput 향상을 달성하였다 (Fig. 10 참조). 이러한 결과는 Atom 이 높은 serving throughput 을 달성하면서도 LLM 을 정확하게 low-bit precision 으로 quantize 할 수 있음을 보여준다.

요약하면, 저자의 기여는 다음과 같다.

  • LLM serving workload 에 대한 포괄적인 성능 분석을 통해 low-bit weight-activation quantization 의 효율성 이점을 규명한다.
  • mixed-precision 과 channel reordering, fine-grained group quantization, dynamic activation quantization, KV-cache quantization 을 결합한 정확한 low-bit weight-activation quantization 알고리즘 Atom 을 제안한다.
  • 효율적인 inference workflow 를 공동 설계하고 low-bit GPU kernel 을 구현한 통합 LLM serving framework 를 제시하며, Atom 의 실질적인 end-to-end throughput 및 latency 를 입증한다.
  • Atom 에 대한 포괄적인 평가를 수행하여, 약간의 accuracy loss 만으로 LLM serving throughput 을 최대 7.7 × 향상시킴을 보인다.

2 Background

Quantization 기법은 high-precision floating point 값을 discrete low-bit 값으로 근사한다. 정수는 uniform range 를 표현하므로, floating point 값을 integer 로 quantize 하는 방식은 단순성과 hardware efficiency 로 인해 널리 사용된다. 일반적인 quantization 은 두 단계로 구성된다.

  • quantization parameter (scale 및 zero point 로 구성됨) 를 결정하는 단계
  • quantized tensor 를 계산하는 단계

uniform asymmetric quantization 의 경우, scale sss 와 zero point zzz 는 다음과 같이 결정된다:

  • ccc 는 clipping factor 로서 outlier 값의 영향을 완화하기 위해 quantization 의 dynamic range 를 줄이는 데 사용된다.

quantization parameter sss 와 zzz 는 calibration data 를 사용하여 static 하게 계산할 수도 있고, inference time 에 runtime statistic 을 사용하여 dynamic 하게 계산할 수도 있다. 이에 따라 quantization 접근은 static 또는 dynamic 으로 분류된다.

LLM 에서는 activation 과 weight matrix 모두에 quantization 을 적용할 수 있으며 (weight-activation quantization), weight 에만 적용할 수도 있다 (weight-only quantization). 그러나 asymmetric weight-activation quantization 은 matrix multiplication 중 추가적인 계산을 유발할 수 있다.

W⋅X=sW(Wˉ−zW)⋅sX(Xˉ−zX).W \cdot X = s_W (\bar{W} - z_W) \cdot s_X (\bar{X} - z_X).W⋅X=sW​(Wˉ−zW​)⋅sX​(Xˉ−zX​).

이 경우 low-bit arithmetic unit 을 사용하기 위해 세 개의 추가 cross-term 을 계산해야 한다. 따라서 efficiency 를 위해 본 연구에서는 symmetric quantization 을 적용한다.

quantization 의 granularity 에 따라 accuracy 와 efficiency 사이에서 서로 다른 trade-off 를 달성할 수 있다.

  • per-tensor quantization: tensor 내 모든 값이 하나의 scale 과 zero point 를 공유한다.
  • per-channel (token) quantization: tensor 의 한 행 또는 한 열에 대해 각각 scale 과 zero point 를 계산한다. 여기서 channel 은 input matrix 의 마지막 dimension 으로 정의한다.
  • per-group quantization: 각 channel 을 여러 sub-group 으로 나누고, 각 group 에 대해 개별적으로 quantization 을 수행한다.

granularity 가 세밀해질수록 quantization precision 은 향상되지만, overhead 또한 증가한다. 본 연구에서는 더 높은 accuracy 를 위해 group quantization 을 채택하며, overhead 를 관리하기 위해 dedicated kernel 을 사용한다 (Sec. 4.2 참조).

3 Performance Analysis of Low-bit LLM Serving

본 절에서는 먼저 serving scenario 에서 LLM inference 의 성능 병목을 분석하고, 이후 low-bit weight-activation quantization 의 중요성을 규명한다. 높은 수요로 인해 LLM serving 은 throughput 중심으로 운영된다. 그러나 LLM inference 의 auto-regressive decode stage 는 한 번에 하나의 token 만을 input 으로 받아 다음 token 을 생성하므로, matrix-vector multiplication (GEMV) 에 의존한다. GEMV 는 큰 weight matrix 를 load 해야 하지만 수행하는 multiplication 수는 적기 때문에, memory-bound 특성을 가진다. 그 결과 GPU utilization 이 낮아지고 compute intensity (computation-to-IO ratio) 가 감소하여 throughput 이 저하된다.

이 문제를 완화하기 위해, 여러 request 의 input 을 결합하여 dense layer (K, Q, V 생성, O projection, MLP) 의 matrix multiplication 을 수행하는 batching 기법이 널리 사용된다. 이를 통해 compute intensity 와 GPU utilization 이 증가한다.

batching 효과를 더욱 활용하고 throughput 을 향상시키기 위해, decode stage 와 prefill stage 의 dense layer input matrix 를 함께 batching 하여 더 큰 matrix 를 구성한다. large batch size 에서는 dense layer 가 compute-bound matrix-matrix multiplication (GEMM) 이 된다.

그러나 decode stage 의 self-attention layer 역시 GEMV 연산이지만, batching 의 이점을 얻지 못한다. 서로 다른 inference request 는 서로 다른 context history 를 가지므로 KV-cache 를 공유하지 않는다. 따라서 request 간 데이터를 묶어 재사용할 수 없으며, 효율성 향상이 발생하지 않는다. FlashAttention 이나 Group Query Attention 과 같은 여러 최적화를 적용하더라도, self-attention layer 는 여전히 KV-cache 의 대규모 memory movement 에 의해 제한된다.

batching 기법을 적용한 이후, 저자는 서로 다른 batch size 에서 operator 별 시간 분해를 측정하였다.

  • Fig. 3 에서 보이듯이, dense layer 와 self-attention layer 모두 throughput 의 병목으로 작용하며 전체 처리 시간의 90% 이상을 소비한다.
  • 이에 따라 저자는 dense layer 와 self-attention layer 모두를 가속하기 위해 quantization mechanism 을 적용한다.

저자는 serving scenario 에서 서로 다른 quantization 접근의 효과를 평가하기 위해 Roofline model 을 사용한다.

Fig. 4(a) 에서 보이듯이,

  • weight-activation quantization 은 효율적인 low-bit hardware arithmetic 덕분에 dense layer 의 compute throughput 을 향상시킨다.
  • 또한 KV-cache 의 크기를 줄여 memory movement 를 감소시킴으로써 self-attention layer 의 throughput 도 증가시킨다.

반면 Fig. 4(b) 에서 보이듯이,

  • weight-only quantization 은 dense layer throughput 을 향상시키지 못한다.
    • 이는 matrix multiplication 이전에 dequantization 이 수행되어야 하며, 연산이 여전히 floating point format 으로 이루어지기 때문이다.
  • 또한 weight-only quantization 은 KV-cache 를 quantize 하지 못하므로, self-attention layer 에 대해서도 이점을 제공하지 못한다.

저자는 Sec. 5 의 Fig. 11(a) 및 Fig. 11(b) 에서 kernel profiling 을 통해 다양한 quantization 기법의 효과를 추가로 정량화한다.

요약하면, serving scenario 에서 throughput 향상 측면에서 low-bit weight-activation quantization 은 weight-only quantization 보다 우수하다. 이는 dense layer 와 self-attention layer 모두를 가속하기 때문이다. 다음 절에서는 Atom 이 low-bit weight-activation quantization 을 통해 높은 accuracy 를 유지하면서도 높은 throughput 을 달성하는 방법을 설명한다.

4 Design

low-bit precision 은 underlying hardware 를 효율적으로 활용하게 하여 throughput 을 향상시킨다. 그러나 low-bit representation 에서 높은 accuracy 를 유지하는 것은 어렵다. LLM 을 극저 bit precision 으로 quantize 하면서 accuracy 를 유지하기 위해, 저자는 LLM 의 특성에 맞춘 일련의 quantization mechanism 을 도입한다. 이러한 mechanism 은 다음을 포함한다.

  • channel reordering 을 포함한 mixed-precision quantization
  • fine-grained group quantization
  • dynamic quantization

이들 기법을 통해 얻는 accuracy 향상은 Tab. 3 의 ablation study 에서 제시된다.

또한 Atom 은 KV-cache 에 대해서도 low-bit quantization 을 적용하여 효율성을 추가로 향상시킨다. 다음 하위 절에서는 각 mechanism 의 세부 사항과 장점을 설명하고, 이어서 end-to-end workflow 를 상세히 기술한다.

4.1 Mixed-precision quantization​

기존 연구는 activation 에 존재하는 outlier 현상이 LLM quantization 의 주요 난점임을 관찰하였다. Fig. 5(a) 에서 보이듯이, 일부 channel 은 다른 channel 에 비해 수 자릿수 이상 큰 magnitude 를 가지며, 이를 outlier 라 한다. 이러한 outlier 의 큰 dynamic range 는 quantization error 를 크게 증가시킨다. 따라서 low-bit quantization 에서 outlier 를 효율적으로 처리하는 것이 중요하다.

이 문제를 완화하는 직관적인 방법은 outlier 와 normal value 를 분리하여 각각 high-bit 와 low-bit 로 quantize 하는 것이다. 이를 mixed-precision 방법이라 한다. Fig. 5(b) 에서 보이듯이, outlier 를 제거한 이후의 나머지 channel 은 훨씬 uniform 해지며, low-bit 값으로 효과적으로 표현될 수 있다. 실험 결과, FP8 및 INT8 과 같은 8-bit representation 은 outlier 표현에 충분하다 (Tab. 3 참조). INT8 은 NVIDIA Tensor Core 와 같은 hardware 에서 널리 지원되므로, Atom 은 outlier 에 대해 INT8 quantization 을 적용한다.

mixed-precision quantization 의 주요 문제는 irregular memory access 이며, 이는 hardware efficiency 를 저하시킨다. regular memory access 를 유지하면서 mixed-precision quantization 을 적용하기 위해, Atom 은 RPTQ 에서 도입된 reordering 기법을 활용한다.

Fig. 7 에서 보이듯이,

  • Atom 은 activation matrix 에서 흩어져 있는 outlier channel 을 matrix 의 끝으로 재정렬한다.
  • 이를 통해 mixed-precision 연산을 효율적으로 구현할 수 있다.
  • 계산 결과의 동등성을 보장하기 위해, weight matrix 역시 activation 의 reorder index 에 맞추어 동일하게 재정렬한다.
  • outlier channel 은 calibration data 를 통해 offline 으로 식별할 수 있으므로, weight matrix 재정렬은 one-time cost 만 발생한다.

반면 activation matrix 의 재정렬은 online 으로 수행되어야 하므로 비용이 클 수 있다.

  • 이를 완화하기 위해, Atom 은 activation matrix reordering operator 를 이전 operator 와 fuse 하여 재정렬 overhead 를 runtime 의 0.5% 이하로 감소시킨다.

4.2 Fine-grained group quantization​

Atom 이 outlier 와 normal value 를 분리하여 quantize 하더라도, 4-bit precision 의 제한된 표현력으로 인해 normal value 를 정확하게 표현하는 것은 여전히 어렵다 (Sec. 5.4 참조). accuracy 를 추가로 향상시키기 위해, group quantization 이 널리 사용된다. 이는 matrix 를 여러 subgroup 으로 나누고, 각 subgroup 내에서 독립적으로 quantization 을 수행하는 방식이다.

예를 들어, group size 가 128 이면, 연속된 128 개 element 를 하나의 group 으로 간주하고 독립적으로 quantize 한다.

group quantization 은 accuracy 향상과 dequantization overhead 사이의 trade-off 를 제공하며, 특히 weight-activation quantization 에서 중요하다. 기존 연구는 GEMM pipeline, 즉 MMA pipeline 에 group dequantization 을 효율적으로 통합하는 방법을 충분히 탐구하지 않았다.

Atom 은 Fig. 8 과 같은 fusion 기법을 제안하여, 실질적인 속도 향상을 제공하는 효율적인 GEMM kernel 을 구현한다 (Sec. 5.3.1 참조).

Atom 의 연산 과정은 다음과 같다.

  • Step 1: activation group 과 대응하는 weight group 의 matrix multiplication 을 수행하여, Tensor Core 와 같은 low-bit hardware 를 사용해 temporary result 를 계산한다.
  • Step 2: 각 temporary result 는 서로 다른 quantization parameter 를 가지므로, CUDA Core 를 사용하여 FP16 representation 으로 dequantize 한다.
  • Step 3: dequantized temporary result 를 더하여 최종 GEMM result 를 얻는다.

fine-grained quantization 으로 인해 각 activation 및 weight group 은 서로 다른 quantization parameter 를 가지므로, dequantization 이 필요하다. overhead 를 관리하기 위해, Atom 은 dequantization 과 summation 을 GEMM kernel, 구체적으로는 MMA pipeline 에 fuse 한다.

이로써 추가 연산은 별도의 memory movement 없이 in-place 로 수행되며, 기존 MMA instruction 과 overlap 되어 실행된다. fused GEMM operator 의 효율성은 Sec. 5.3.1 에서 제시된다.

group size 가 128 이고 high precision channel size 가 128 인 경우, Llama-7B 에서 Atom 의 effective bit 는 4.251 이다. effective bit 는 quantization parameter 를 포함하여 각 element 당 평균 사용 bit 수로 정의된다. 이 지표는 weight-only quantization 연구에서 실제 compression ratio 및 memory-bound 환경에서의 속도 향상을 반영하기 위해 널리 사용된다.

그러나 serving scenario 에서 weight-activation quantization 의 주요 이점은 memory reduction 이 아니라 low-bit arithmetic unit 활용에 따른 computation efficiency 이다. 따라서 이후 논의에서는 이 지표를 사용하지 않는다.

4.3 Dynamic quantization process​

fine-grained quantization 은 activation 의 각 channel 내부의 local variation 을 더 잘 보존할 수 있다. 그러나 calibration data 를 기반으로 quantization parameter 를 static 하게 계산할 경우, 실제 input 이 서로 다른 local distribution 을 가질 수 있으므로 이러한 장점이 감소할 수 있다.

따라서 Atom 은 inference 중 각 activation matrix 에 대해 quantization parameter 를 맞춤 설정하는 dynamic quantization 을 채택한다. dynamic quantization 의 overhead 를 완화하기 위해, Atom 은 ZeroQuant 와 유사하게 quantization 연산을 이전 operator 에 fuse 한다. 추가되는 operator 는 reduction 과 element-wise division 을 포함하는 element-wise 연산이므로, Fig. 3 에서 보이듯이 시간 소모가 큰 dense layer 및 self-attention layer 에 비해 fused operator 의 runtime 은 여전히 미미하다.

그러나 asymmetric quantization 은 상당한 추가 computation 으로 인해 runtime overhead 를 크게 증가시킬 수 있다 (Sec. 2 참조). throughput 과 accuracy 사이의 균형을 맞추기 위해, Atom 은 신중하게 선택된 clip threshold 와 함께 symmetric quantization 을 사용한다. 또한 weight matrix 를 quantize 할 때 GPTQ 를 적용한다. 이는 완전히 offline process 이므로 runtime efficiency 를 저하시키지 않으면서 accuracy 를 향상시킨다.

4.4 KV-cache quantization​

Sec. 3 에서 설명했듯이, decode stage 의 self-attention layer 는 매우 memory-bound 특성을 가진다. 이를 완화하기 위해 Atom 은 KV-cache 에도 low-bit quantization 을 적용한다. Atom 은 low-bit precision 으로 KV-cache 를 load 한 뒤, FP16 계산을 수행하기 직전에 dequantize 한다. 이를 통해 대규모 memory reduction 이 이루어지며 throughput 이 크게 향상된다.

한편 asymmetric quantization 과 symmetric quantization 은 KV-cache 의 memory movement 측면에서 유사하므로, memory-bound self-attention layer 에서의 성능도 유사하다. 따라서 Atom 은 accuracy 이점을 제공하는 asymmetric quantization 을 KV-cache 에 적용한다.

activation matrix 와 비교할 때, KV-cache 는 quantization 에 더 적합하다고 볼 수 있다.

  • self-attention 연산에서는 들어오는 token 의 Query vector 가 K cache 와 곱해진다.
  • 그 결과는 Softmax 로 정규화된 후, V cache 와 다시 곱해져 최종 output 을 생성한다.
  • Softmax 의 정규화 효과로 인해 K cache 의 quantization error 는 output 에 미치는 영향이 상대적으로 작다.
  • 또한 Fig. 9 의 profiling 결과에 따르면 V cache 는 outlier 현상이 덜 빈번하게 나타나며, 이는 quantization 에 더 적합함을 의미한다.

따라서 Atom 은 attention head 단위의 granularity 로 asymmetric low-bit quantization 을 직접 적용하며, Sec. 5.4 에서 보이듯이 높은 accuracy 를 유지한다.

4.5 Implementation of quantization workflow​

저자는 제안한 설계의 실현 가능성을 보이기 위해 Llama model 에 Atom 을 구현하였다 (Fig. 6 참조).

quantization 의 이점을 극대화하기 위해, Atom 은 추가 operator 의 overhead 를 kernel fusion 을 통해 관리한다. 구체적으로, reordering, quantization, dequantization 을 포함한 quantization operator 를 기존 operator 에 fuse 한다.

compute-bound dense layer 에 대해서는 low-bit unit 을 활용하여 throughput 을 향상시킨다. memory-bound self-attention layer 에 대해서는 LLM serving 을 위한 kernel library 인 FlashInfer 와 dequantization 을 fuse 하여, KV-cache 에서 low-bit 값만 load 되도록 한다. 또한 Atom 은 large batch size 를 가능하게 하기 위해 효율적인 memory 사용 기법인 PageAttention 을 통합한다.

5 Evaluation

저자는 Atom 의 accuracy 와 efficiency 에 대해 포괄적인 평가를 수행한다.

  • accuracy 평가는 generation perplexity 및 zero-shot accuracy 와 같은 널리 사용되는 metric 으로 수행한다.
  • efficiency 평가는 per-kernel 성능 분석에서 시작하여 end-to-end throughput 및 latency 평가로 확장한다.

또한 다양한 기법이 Atom 에 미치는 영향을 이해하기 위해 ablation study 를 수행하며, 이를 통해 각 설계 선택이 가지는 efficiency 와 accuracy 사이의 trade-off 를 분석한다.

5.1 Quantization setup​

Atom 은 weight 와 activation 에 대해 symmetric quantization 을 적용하고, KV-cache 에 대해서는 asymmetric quantization 을 적용한다. group size 는 128 로 설정하여 평가한다.

outlier channel 식별을 위해, 기존 연구를 따라 WikiText2 에서 무작위로 샘플링한 128 개 문장을 calibration data 로 사용한다. 각 channel 의 square sum 값이 가장 높은 128 개 channel 을 outlier 로 선택하고, 이를 INT8 로 유지한다. 이후 activation 및 weight matrix 를 outlier channel 의 index 에 따라 재정렬한다.

재정렬 이후, weight matrix 에 대해 GPTQ 를 적용한다. clipping factor 는 grid search 를 통해 activation 에 대해 0.9, weight 에 대해 0.85 를 각각 최적 값으로 설정한다.

weight quantization 및 outlier 식별을 위한 preprocessing 은 단일 RTX Ada 6000 GPU 에서 layer-by-layer 로 수행한다. Llama-65B 의 경우 전체 과정에 약 4 시간 이 소요된다.

5.2 Accuracy evaluation​

Benchmarks​

저자는 공개된 Llama model 을 대상으로 Atom 을 평가한다. INT4 및 INT3 weight-activation quantization 과 같은 low-bit 설정에 초점을 맞춘다.

accuracy metric 으로 perplexity 와 zero-shot accuracy 를 사용한다.

  • perplexity 평가는 WikiText2, PTB, C4 dataset 에서 수행한다.
  • zero-shot task 평가는 lm-eval 기반으로 PIQA, ARC, BoolQ, HellaSwag, WinoGrande task 에서 수행한다.

Baselines​

Atom 은 다음과 같은 post-training quantization 기법과 비교된다.

  • SmoothQuant
  • OmniQuant
  • QLLM

SmoothQuant 는 공식 코드가 Llama model 을 지원하지 않으므로 자체 구현을 사용하며, W8A8 quantization 만 지원한다. alpha 값에 대해 grid search 를 수행하여 benchmark 별 최적 값을 보고한다.

OmniQuant 는 W4A4 평가에 대해 사전 quantize 된 weight 를 사용하고, W3A3 은 공식 코드를 실행하여 평가한다. W3A3 의 최적 성능을 위해 learning rate 1e−41e^{-4}1e−4 및 alpha 0.750.750.75 를 사용한다. Llama-30B 및 Llama-65B 에 대해서는 resource 요구가 커 W3A3 OmniQuant 를 생략한다.

QLLM 은 논문에 보고된 W4A4 결과만 사용하며, 실험 당시 코드가 공개되지 않아 W3A3 은 평가하지 않는다.

Zero-shot accuracy​

Tab. 1 은 Llama model 에 대해 6 개 task 의 zero-shot accuracy 를 비교한 결과를 보여준다.

Atom 은 다른 weight-activation quantization 방법보다 현저히 우수한 성능을 보인다. W4A4 설정에서 FP16 대비 평균 accuracy loss 는 다음과 같다.

  • 7B: 2.3%
  • 13B: 1.7%
  • 30B: 0.4%
  • 65B: 1.4%

반면 기존 방법은 동일 설정에서 9.6% ~ 23.8% 의 accuracy loss 를 보인다.

Perplexity​

Tab. 2 는 Llama model 에 대한 perplexity 결과를 보고한다.

  • OmniQuant 및 QLLM 은 W4A4 설정에서 perplexity 를 약 10 수준까지 낮추었으나, 여전히 accuracy loss 가 크다.
  • Atom 은 perplexity 를 더욱 감소시켜 Llama-65B 에서 세 dataset 모두에 대해 0.4 이하의 perplexity 증가만을 보인다.
  • W3A3 설정에서도 Atom 은 Llama-65B 에 대해 평균 2.3 의 perplexity 증가만을 보이며, 기존 방법은 수용 가능한 perplexity 를 달성하지 못한다.
  • 또한 큰 model 일수록 Atom 의 accuracy loss 가 더 작게 나타난다.

5.3 Efficiency evaluation​

Atom 의 efficiency 를 입증하기 위해 per-kernel 성능과 end-to-end 성능을 모두 profiling 한다.

NVIDIA GPU 는 INT4 arithmetic 을 효율적으로 지원하므로, 24GB RTX 4090 (CUDA 11.3) 에서 W4A4 설정으로 평가한다.

5.3.1 Kernel evaluation​

Matrix multiplication​

Fig. 11(a) 는 Atom 이 구현한 fused GEMM operator 의 성능을 보여준다.

baseline 으로 다음을 구현하였다.

  • W8A8 fused GEMM
  • W4A16 (4-bit weight-only) fused GEMM

작은 batch size 에서는 GEMM 이 memory-bound 이므로 weight-only quantization 의 memory reduction 이 효과적이다.

그러나 batch size 가 증가하면 compute-bound 환경이 되며, FP16 계산이 필요한 weight-only quantization 의 효율은 감소한다.

반면 4-bit Atom 은 hardware efficiency 덕분에 모든 방법을 능가한다. batch size 512 에서 Atom 은 FP16 대비 3.4×, INT8 대비 1.9× 의 matrix multiplication 속도 향상을 달성한다.

Self-attention​

self-attention layer 에 대해서는 FlashInfer kernel library 에 quantization 을 fuse 하였다. 또한 효율적인 memory 사용을 위해 PageAttention 을 통합하였다.

Fig. 11(b) 에서 보이듯이, bit 수 감소는 KV-cache memory 사용량을 선형적으로 줄이며, memory-bound 환경에서 throughput 을 비례적으로 증가시킨다.

batch size 128 에서 Atom 은 INT8 대비 1.8×, FP16 대비 3.5× 의 속도 향상을 달성한다.

5.3.2 End-to-end evaluation​

Serving setup​

Atom 은 LLM serving framework 인 Punica 에 통합되어 end-to-end 성능을 평가한다.

baseline 으로 W8A8 및 W4A16 을 통합하였다. representative workload 생성을 위해 ShareGPT 데이터의 prefill 및 decode request length 분포를 사용한다. multi-round conversation 은 여러 사용자 request 로 간주하며, 이전 prompt 와 response 를 모두 concatenation 하여 새로운 request 의 prompt 로 사용한다.

batch size 는 8 에서 256 까지 변화시킨다. 모든 request 는 First-Come-First-Served 방식으로 처리한다. request 가 종료되면 Orca 에서 제안된 continuous batching 방식으로 새로운 request 를 즉시 batch 에 추가한다.

GPU memory 제한으로 인해, 큰 batch size 에 대해서는 작은 batch size 의 KV-cache 를 재사용하여 data access pattern 과 computation 양을 유지한 상태로 성능을 simulation 한다.

End-to-end throughput​

Fig. 10(a) 는 초당 생성 token 수를 나타내는 end-to-end throughput 을 보여준다. 실선은 실제 측정 결과, 점선은 simulation 결과이다.

  • 모든 batch size 에서 Atom 은 다른 quantization 방법을 능가한다.
  • Fig. 10(c) 와 같이 memory 를 고정하면, Atom 은 더 큰 batch size 를 지원하여 latency target 을 유지하면서 baseline 을 더욱 크게 초과하는 throughput 을 달성한다.
  • Atom 은 동일 memory 사용량 기준으로 FP16 대비 7.73×, INT8 대비 2.53× 의 throughput 향상을 달성한다.
  • 반면 weight-only quantization 은 dense layer 의 FP16 계산 한계와 self-attention layer 의 KV-cache memory movement 에 의해 제한된다.

End-to-end latency​

latency 는 queuing time 을 제외한 token 당 평균 decoding 시간으로 측정한다.

모든 batch size 에서 Atom 은 다른 quantization 방법보다 우수하다. batch size 64 에서 최고 실용 성능을 달성하며, 이때 latency 는 batch size 8 의 INT8 또는 FP16 구현보다도 낮다.

또한 batch size 256 에서도 latency 는 100 ms 미만으로 유지되며, 이는 인간의 눈이 인지할 수 있는 효과적인 reading speed 로 보고된 값보다 낮다.

5.4 Ablation study of quantization techniques​

본 절에서는 Atom 에서 사용된 quantization 기법들의 효과를 accuracy 와 efficiency 측면에서 종합적으로 평가한다. 이를 통해 설계 선택의 타당성과 accuracy–efficiency trade-off 를 명확히 한다.

5.4.1 Ablation study to evaluate accuracy​

저자는 Atom 에서 사용된 각 quantization 기법이 accuracy 에 미치는 영향을 분석한다.

먼저 RTN 을 사용하고, weight 에는 per-channel quantization, activation 에는 per-token quantization 을 적용하는 표준 quantization recipe 로 model 을 W4A4 로 quantize 한다. 이후 Atom 에서 사용하는 다음 기법을 순차적으로 적용하면서 perplexity 변화를 분석한다.

  • mixed-precision
  • outlier quantization
  • group quantization
  • clipping
  • GPTQ
  • KV-cache quantization

Tab. 3 에서 보이듯이,

  • outlier channel 을 FP16 으로 유지하면 perplexity 가 크게 감소한다.
  • 이후 outlier 를 INT8 로 quantize 하더라도 perplexity 는 0.05 만 증가하며, 이는 mixed precision 이 outlier 문제를 효과적으로 해결함을 보여준다.
  • 또한 fine-grained group quantization 은 또 한 번의 큰 perplexity 감소를 가져온다. clipping 과 GPTQ 는 각각 0.09 의 perplexity 감소를 추가로 제공한다.
  • 마지막으로 KV-cache 를 quantize 하면 perplexity 가 0.12 증가하는데, 이는 Sec. 4.4 에서의 분석과 일치한다.

5.4.2 Ablation study to evaluate efficiency​

다음으로 서로 다른 fused quantization 기법이 GEMM kernel throughput 에 미치는 영향을 분석한다.

quantization 연산이 없는 pure INT4 GEMM 구현은 약 980 TOPS 를 달성한다. mixed precision fusion (128 개 channel 을 INT8 Tensor Core 로 유지) 을 적용하면 약 8% overhead 가 발생하여 900 TOPS 의 throughput 을 보인다.

fine-grained group quantization 은 compute pipeline 에 깊게 영향을 미치므로 가장 큰 overhead 를 유발한다. group dequantization 을 fusion 하면 성능은 770 TOPS 로 감소한다. 그럼에도 fused GEMM kernel 은 INT8 의 theoretical throughput 한계를 약 18% 초과한다.

또한 channel reordering 의 효율성을 검증하기 위해 Atom 과 baseline 을 비교하는 ablation study 를 수행한다. baseline 은 mixed precision 을 위해 matrix decomposition 을 사용하는 기존 방식으로 구현된다.

반면 Atom 은 reordering 및 quantization 을 포함한 operator 를 기존 operator 에 fuse 한다. batch size 16 에서 256 까지 평가하여 layer norm 과 GEMM 연산의 inference latency 를 측정한 결과, Atom 은 baseline 대비 25% 에서 35% 의 성능 향상을 일관되게 달성한다.

6 Discussion

Mixture of Experts (MoE), State Space Models (SSMs) 과 같은 새로운 model architecture 의 등장과 NVIDIA Blackwell GPU 와 같은 hardware accelerator 의 발전에 따라, Atom 이 새로운 model 과 hardware 에도 적용 가능한지 검증하는 것이 중요하다. 본 절에서는 추가 LLM 및 data format 에 대한 평가를 제공한다.

Generality on models​

Atom 이 높은 accuracy 를 달성하는 핵심 기법은 outlier 에 대한 mixed precision 과 normal value 에 대한 fine-grained quantization 이다. 저자는 이러한 기법이 최신 transformer 기반 LLM 에도 일반화 가능함을 실험적으로 확인한다.

Tab. 4 는 Llama-2 및 Mixtral model 에 대한 perplexity 결과를 보여준다.

MoE model 에 일반화하기 위해, Atom 은 expert 별 FFN 에 대해 서로 다른 reorder index 를 사용하도록 조정하면 된다.

Tab. 4 에서 보이듯이, Atom 은 baseline 대비 우수한 성능을 유지하면서 높은 accuracy 를 달성한다.

Generality on data formats​

FP4 및 MX 와 같은 새로운 data format 이 hardware 에서 지원됨에 따라, Atom 을 FP4 에 대해서도 평가한다. Tab. 4 에서 보이듯이, weight 와 activation 을 모두 FP4 로 quantize 하더라도 INT4 와 유사한 accuracy 를 유지한다. 이는 INT4 와 FP4 의 representation capability 가 유사함을 시사한다.

또한 NVIDIA Blackwell GPU 는 MX format 에서 group quantization 을 지원한다. 이러한 hardware 기능은 Sec. 5.4.2 에서 설명한 Atom 의 group quantization overhead 를 완화할 것으로 기대된다.

7 Related Work

LLM serving​

LLM serving throughput 을 개선하기 위한 다양한 연구가 수행되었다.

  • batching 효과를 분석한 연구
  • continuous batching 을 제안한 Orca
  • page table 기반 KV-cache 관리를 제안한 vLLM
  • offload mechanism 으로 대규모 batch 를 지원한 FlexGen

그러나 기존 연구와 달리, 본 연구는 quantization 과 LLM serving 의 교차 지점을 심층적으로 탐구한다.

Weight-only quantization​

LLM 에서 weight matrix 는 큰 memory movement 를 유발하여 decode 효율을 제한한다. weight-only quantization 은 weight matrix 를 low-bit precision 으로 근사한다.

  • GPTQ 는 approximate second-order information 을 사용하여 4-bit quantization 을 수행한다.
  • AWQ 는 salient weight 를 보존하여 accuracy 를 향상시킨다.
  • SqueezeLLM 은 non-uniform quantization 과 sparse format 으로 outlier 를 처리한다.
  • QuiP 는 adaptive rounding 으로 2-bit representation 을 달성한다.

그러나 serving scenario 에서는 batching 으로 weight load overhead 가 amortize 되어 dense layer 가 compute-bound 가 된다. weight-only quantization 은 low-bit hardware 를 충분히 활용하지 못하여 이상적인 throughput 을 제공하지 못한다.

Weight-activation quantization​

weight-activation quantization 은 weight 와 activation 모두를 quantize 하며, activation 의 outlier 현상으로 인해 더 어렵다.

  • LLM.INT8 는 mixed precision 으로 activation outlier 를 보존한다.
  • 여러 연구는 activation outlier 를 수학적 변환으로 처리하였다.
  • RPTQ 는 channel 재배열로 quantization group 내 분산을 줄였다.
  • 일부 연구는 low-rank matrix 로 quantization error 를 보정하였다.
  • 다른 연구는 algorithm–architecture co-design 을 활용하였다.

그러나 이들 방법은 극저 bit precision 에서 accuracy 손실이 크거나, practical hardware support 가 부족하다. 본 연구의 방법은 low-bit representation 에서도 높은 accuracy 를 달성하며 practical speedup 을 보장한다.

8 Conclusion

저자는 LLM serving 에서 높은 accuracy 와 높은 throughput 을 동시에 달성하기 위해 underlying hardware 를 효율적으로 활용하는 low-bit quantization 방법 Atom 을 제안하였다.

Atom 은 다음 기법을 결합한다.

  • reordering 을 포함한 mixed-precision quantization
  • fine-grained group quantization
  • dynamic quantization
  • KV-cache quantization

이를 통해 emerging low-bit hardware support 를 최대한 활용하면서 accuracy 를 유지한다. Atom 을 end-to-end serving framework 에 통합한 결과, FP16 baseline 대비 최대 7.73× 의 throughput 향상을 달성하였으며, zero-shot accuracy loss 는 1.4% 미만으로 유지하였다.
