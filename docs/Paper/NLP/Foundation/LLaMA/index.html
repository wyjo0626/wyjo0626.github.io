LLaMA: Open and Efficient Foundation Language Models

논문 및 이미지 출처 : https://arxiv.org/pdf/2302.13971

Abstract

저자는 7B 에서 65B parameters 까지 범위를 가지는 foundation language models 모음인 LLaMA 를 소개한다.

저자는 저자의 model 을 trillions 개의 tokens 로 training 하며, proprietary 하고 접근할 수 없는 dataset 에 의존하지 않고도 오직 publicly available datasets 만 사용하여 state-of-the-art model 을 training 할 수 있음을 보인다.

특히, LLaMA-13B 는 대부분의 benchmarks 에서 GPT-3 (175B) 를 능가하며, LLaMA-65B 는 최고 수준의 model 인 Chinchilla-70B 와 PaLM-540B 에 견줄 만하다. 저자는 모든 model 을 research community 에 공개한다.

1 Introduction

방대한 text corpora 에서 training 된 Large Languages Models (LLMs) 은 textual instructions 또는 소수의 examples 로부터 새로운 tasks 를 수행할 수 있는 능력을 보여주었다. 이러한 few-shot 성질은 model 을 충분히 큰 크기로 scaling 했을 때 처음 나타났으며, 그 결과 이러한 model 을 더욱 scaling 하는 데 초점을 맞춘 연구 흐름이 형성되었다.

이러한 노력은 더 많은 parameters 가 더 나은 performance 로 이어진다는 가정에 기반한다. 그러나 Hoffmann et al. 은 최근 연구에서, 주어진 compute budget 에 대해 최고의 performance 는 가장 큰 model 이 아니라 더 많은 data 로 training 된 더 작은 model 에 의해 달성된다는 점을 보였다.

  • Hoffmann et al. 의 scaling laws 의 목적은 특정 training compute budget 에 대해 dataset 크기와 model 크기를 어떻게 가장 잘 scaling 할지를 결정하는 것이다.
    • 그러나 이 목적은 inference budget 을 고려하지 않으며, language model 을 대규모로 serving 할 때 이는 결정적으로 중요해진다.
    • 이러한 맥락에서, 주어진 target performance 수준에 대해 선호되는 model 은 training 이 가장 빠른 model 이 아니라 inference 가 가장 빠른 model 이다.
  • 또한 일정 수준의 performance 에 도달하기 위해 큰 model 을 training 하는 것이 더 저렴할 수 있지만, 더 작은 model 을 더 오래 training 하는 편이 궁극적으로 inference 에서는 더 저렴해진다.
    • 예를 들어, Hoffmann et al. 은 10B model 을 200B tokens 로 training 할 것을 권장하지만, 저자는 7B model 의 performance 가 1T tokens 이후에도 계속 향상된다는 점을 발견한다.

이 연구의 초점은 일반적으로 사용되는 것보다 더 많은 tokens 로 training 함으로써, 다양한 inference budgets 에서 가능한 한 최고의 performance 를 달성하는 일련의 language models 를 training 하는 것이다. 그 결과 얻어진 model 인 LLaMA 는 7B 에서 65B parameters 범위를 가지며, 기존 최고의 LLMs 와 비교해도 경쟁력 있는 performance 를 보인다.

  • 예를 들어, LLaMA-13B 는 GPT-3 보다 10× 더 작음에도 대부분의 benchmarks 에서 GPT-3 를 능가한다.
  • 저자는 이 model 이 single GPU 에서 실행될 수 있기 때문에 LLMs 에 대한 접근과 연구를 democratize 하는 데 도움이 될 것이라고 믿는다.
  • 더 큰 규모 측면에서, 저자의 65B-parameter model 역시 Chinchilla 나 PaLM-540B 와 같은 최고 수준의 large language models 와 경쟁력이 있다.

Chinchilla, PaLM, GPT-3 와 달리, 저자는 오직 publicly available data 만 사용하므로 저자의 연구는 open-sourcing 과 양립 가능하다. 반면 대부분의 기존 model 은 publicly available 하지 않거나 문서화되지 않은 data (e.g., “Books – 2TB” 또는 “Social media conversations”) 에 의존한다. 일부 예외로는 OPT, GPT-NeoX, BLOOM, GLM 이 존재하지만, 이들 중 PaLM-62B 또는 Chinchilla 와 경쟁력 있는 것은 없다.

이 논문의 나머지 부분에서 저자는 transformer architecture 에 가한 수정 사항들과 training method 의 개요를 제시한다. 이어서 저자는 저자의 model 성능을 보고하고, 표준 benchmarks 집합에서 다른 LLMs 와 비교한다. 마지막으로, responsible AI community 의 최신 benchmarks 일부를 사용하여 저자의 model 에 내재된 bias 와 toxicity 를 드러낸다.

2 Approach

저자의 training approach 는 이전 연구에서 설명된 방법과 유사하며, Chinchilla scaling laws 에서 영감을 받았다. 저자는 standard optimizer 를 사용하여 대량의 textual data 위에서 large transformers 를 training 한다.

2.1 Pre-training Data​

저자의 training dataset 은 다양한 domains 를 포괄하는 여러 source 의 mixture 로 구성되며, 이는 Tab. 1 에 보고되어 있다.

  • 대부분의 경우 저자는 다른 LLMs 를 training 하는 데 활용된 data sources 를 재사용하되, 오직 publicly available 하고 open sourcing 과 양립 가능한 data 만 사용한다는 제한을 둔다.
  • 이는 다음과 같은 data mixture 와 각 data 가 training set 에서 차지하는 비율로 이어진다.

English CommonCrawl [67%].​

저자는 2017 년부터 2020 년까지의 CommonCrawl dumps 5 개를 CCNet pipeline 으로 preprocess 한다.

  • 이 과정은 line level 에서 data deduplication 을 수행하고, fastText linear classifier 로 language identification 을 수행하여 non-English pages 를 제거하며, n-gram language model 로 low quality content 를 filtering 한다.
  • 추가로, 저자는 Wikipedia 에서 references 로 사용된 pages 와 무작위로 sampling 된 pages 를 분류하는 linear model 을 training 하였고, references 로 분류되지 않은 pages 는 폐기하였다.

C4 [15%].​

탐색적 experiments 동안, 저자는 다양한 pre-processed CommonCrawl datasets 를 사용하는 것이 performance 를 향상시킨다는 점을 관찰하였다. 따라서 저자는 publicly available 한 C4 dataset 을 data 에 포함하였다. C4 의 preprocessing 역시 deduplication 과 language identification 단계를 포함한다. CCNet 과의 주요 차이는 quality filtering 에 있으며, C4 는 주로 punctuation marks 의 존재 여부나 webpage 내 단어 및 문장 수와 같은 heuristics 에 의존한다.

Github [4.5].​

저자는 Google BigQuery 에서 제공되는 public GitHub dataset 을 사용한다. 저자는 Apache, BSD, MIT licenses 하에 배포된 projects 만 유지하였다. 또한, line length 또는 alphanumeric characters 비율에 기반한 heuristics 로 low quality files 를 filtering 하였고, regular expressions 로 headers 와 같은 boilerplate 를 제거하였다. 마지막으로, 저자는 exact matches 를 사용하여 file level 에서 결과 dataset 을 deduplicate 한다.

Wikipedia [4.5%].​

저자는 2022 년 6 월부터 8 월 사이 기간의 Wikipedia dumps 를 추가하며, 여기에는 Latin 또는 Cyrillic scripts 를 사용하는 20 개 언어, 즉 bg, ca, cs, da, de, en, es, fr, hr, hu, it, nl, pl, pt, ro, ru, sl, sr, sv, uk 가 포함된다. 저자는 hyperlinks, comments, 기타 formatting boilerplate 를 제거하도록 data 를 처리한다.

Gutenberg and Books3 [4.5%].​

저자는 training dataset 에 두 개의 book corpora 를 포함한다. 하나는 public domain 에 속하는 books 를 포함하는 Gutenberg Project 이고, 다른 하나는 large language models training 을 위한 publicly available dataset 인 ThePile 의 Books3 section 이다. 저자는 book level 에서 deduplication 을 수행하며, 내용 overlap 이 90% 를 초과하는 books 를 제거한다.

ArXiv [2.5].​

저자는 scientific data 를 dataset 에 추가하기 위해 arXiv Latex files 를 처리한다. Lewkowycz et al. 을 따라, 저자는 첫 번째 section 이전의 모든 내용과 bibliography 를 제거하였다. 또한 저자는 .tex files 에서 comments 를 제거하고, papers 간 consistency 를 높이기 위해 사용자가 작성한 definitions 와 macros 를 inline-expanded 하였다.

Stack Exchange [2%].​

저자는 다양한 domains, 즉 computer science 에서 chemistry 에 이르는 high quality questions and answers 웹사이트인 Stack Exchange dump 를 포함한다. 저자는 가장 큰 28 개 websites 의 data 를 유지하고, text 에서 HTML tags 를 제거하며, answers 를 score 순서대로 (가장 높은 점수에서 가장 낮은 점수로) 정렬하였다.

Tokenizer​

저자는 SentencePiece 의 implementation 을 사용하여 byte-pair encoding (BPE) algorithm 으로 data 를 tokenize 한다. 특히 저자는 모든 숫자를 개별 digits 로 분할하고, 알 수 없는 UTF-8 characters 를 분해하기 위해 bytes 로 fallback 한다.

전체적으로, 저자의 전체 training dataset 은 tokenization 이후 대략 1.4T tokens 를 포함한다. 저자의 대부분의 training data 에 대해 각 token 은 training 동안 한 번만 사용되며, Wikipedia 와 Books domains 에 대해서만 대략 두 epochs 를 수행한다.

2.2 Architecture​

최근 large language models 연구를 따라, 저자의 network 는 transformer architecture 에 기반한다. 저자는 이후 제안되었고 PaLM 과 같은 다양한 model 에 사용된 여러 개선점을 활용한다. 다음은 original architecture 와의 주요 차이점과, 각 변경에 대해 영감을 얻은 출처이다.

Pre-normalization [GPT3].​

training stability 를 향상시키기 위해, 저자는 output 을 normalize 하는 대신 각 transformer sub-layer 의 input 을 normalize 한다. 저자는 Zhang and Sennrich 가 도입한 RMSNorm normalizing function 을 사용한다.

SwiGLU activation function [PaLM].​

performance 향상을 위해 저자는 ReLU non-linearity 를 Shazeer 가 도입한 SwiGLU activation function 으로 대체한다. 저자는 PaLM 의 4d4d4d 대신 234d\frac{2}{3} 4d32​4d 의 dimension 을 사용한다.

Rotary Embeddings [GPTNeo].​

저자는 absolute positional embeddings 를 제거하고, 대신 Su et al. 이 도입한 rotary positional embeddings (RoPE) 를 network 의 각 layer 에 추가한다.

저자의 서로 다른 model 에 대한 hyper-parameters 세부 사항은 Tab. 2 에 제시되어 있다.

2.3 Optimizer​

저자의 model 은 AdamW optimizer 를 사용하여 training 되며, hyper-parameters 는 다음과 같다:

  • β1=0.9\beta_1 = 0.9β1​=0.9, β2=0.95\beta_2 = 0.95β2​=0.95. 저자는 cosine learning rate schedule 을 사용하며, 최종 learning rate 는 maximal learning rate 의 10% 와 같아지도록 한다.
  • 저자는 weight decay 0.1 과 gradient clipping 1.0 을 사용한다.
  • 저자는 2,000 warmup steps 를 사용하며, model 크기에 따라 learning rate 와 batch size 를 조정한다 (세부 사항은 Tab. 2 참조).

2.4 Efficient implementation​

저자는 model 의 training speed 를 향상시키기 위해 여러 최적화를 수행한다.

  • 먼저, memory usage 와 runtime 을 줄이기 위해 causal multi-head attention 의 efficient implementation 을 사용한다.
  • xformers library 에 제공되는 이 implementation 은 Rabe and Staats 에서 영감을 받았으며, Dao et al. 의 backward 를 사용한다.
  • 이는 attention weights 를 저장하지 않고, language modeling task 의 causal nature 때문에 masked 되는 key/query scores 를 계산하지 않음으로써 달성된다.

training efficiency 를 더욱 향상시키기 위해, 저자는 checkpointing 을 사용하여 backward pass 동안 recompute 되는 activations 의 양을 줄였다.

  • 보다 정확히는, 저자는 linear layers 의 outputs 와 같이 계산 비용이 큰 activations 를 저장한다.
  • 이는 PyTorch autograd 에 의존하는 대신 transformer layers 에 대한 backward function 을 수동으로 구현함으로써 달성된다.

이 최적화를 충분히 활용하기 위해, 저자는 Korthikanti et al. 에서 설명한 바와 같이 model parallelism 과 sequence parallelism 을 사용하여 model 의 memory usage 를 줄일 필요가 있다. 또한 저자는 activations 계산과 network 를 통한 GPUs 간 communication (all_reduce operations 로 인한) 을 가능한 한 많이 overlap 한다.

65B-parameter model 을 training 할 때, 저자의 code 는 80GB RAM 을 가진 2048 개의 A100 GPU 에서 약 380 tokens/sec/GPU 를 처리한다. 이는 1.4T tokens 를 포함하는 저자의 dataset 전체에 대한 training 이 대략 21 일이 걸린다는 것을 의미한다.

3 Main results

이전 연구를 따라, 저자는 zero-shot 과 few-shot tasks 를 고려하며, 총 20 개 benchmark 에 대한 결과를 보고한다.

  • Zero-shot. 저자는 task 에 대한 textual description 과 test example 을 제공한다. model 은 open-ended generation 을 사용하여 답을 제공하거나, 제안된 답들을 ranking 한다.
  • Few-shot. 저자는 task 의 몇 개 examples (1 개에서 64 개 사이) 와 test example 을 제공한다. model 은 이 text 를 input 으로 받아 답을 생성하거나 서로 다른 options 을 ranking 한다.

저자는 LLaMA 를 다른 foundation models 와 비교하는데, 여기에는 publicly available 하지 않은 language models 인 GPT-3, Gopher, Chinchilla, PaLM 과, open-sourced 된 OPT models, GPT-J, GPT-Neo 가 포함된다. 또한 Sec. 4 에서 저자는 OPT-IML 및 Flan-PaLM 과 같은 instruction-tuned models 와도 LLaMA 를 간단히 비교한다.

저자는 free-form generation tasks 와 multiple choice tasks 에서 LLaMA 를 평가한다.

  • multiple choice tasks 에서의 목적은 주어진 context 를 바탕으로, 주어진 options 집합 중 가장 적절한 completion 을 선택하는 것이다.
  • 저자는 제공된 context 가 주어졌을 때 likelihood 가 가장 높은 completion 을 선택한다. 저자는 Gao et al. 을 따르며 completion 의 characters 수로 normalize 된 likelihood 를 사용한다.
  • 단, 일부 datasets (OpenBookQA, BoolQ) 에 대해서는 Brown et al. 을 따르며, “Answer:” 를 context 로 주었을 때의 completion likelihood 로 normalize 된 likelihood 에 기반하여 completion 을 선택한다:
P(completion∣context)/P(completion∣"Answer:")P(\text{completion}|\text{context}) / P(\text{completion}|\text{"Answer:"})P(completion∣context)/P(completion∣"Answer:")

3.1 Common Sense Reasoning​

저자는 8 개의 표준 common sense reasoning benchmarks 를 고려한다: BoolQ, PIQA, SIQA, HellaSwag, WinoGrande, ARC easy, ARC challenge, OpenBookQA. 이 datasets 는 Cloze 및 Winograd style tasks 와 multiple choice question answering 을 포함한다. 저자는 language modeling community 에서 수행된 방식대로 zero-shot setting 에서 평가한다.

Tab. 3 에서 저자는 다양한 크기의 기존 models 와 비교하며, 해당 논문들에 보고된 수치를 제시한다.

  • 첫째, LLaMA-65B 는 BoolQ 를 제외한 보고된 모든 benchmarks 에서 Chinchilla-70B 를 능가한다.
  • 유사하게, 이 model 은 BoolQ 와 WinoGrande 를 제외한 모든 곳에서 PaLM-540B 를 능가한다.
  • LLaMA-13B model 역시 GPT-3 보다 10× 더 작음에도 대부분의 benchmarks 에서 GPT-3 를 능가한다.

3.2 Closed-book Question Answering​

저자는 두 개의 closed-book question answering benchmarks 인 Natural Questions 와 TriviaQA 에서 LLaMA 를 기존 large language models 와 비교한다. 두 benchmarks 모두에 대해, 저자는 closed book setting 에서 exact match performance 를 보고한다. 즉, model 이 질문에 답하기 위한 evidence 를 포함한 documents 에 접근하지 못하는 setting 이다.

  • Tab. 4 에서는 Natural Questions 에 대한 performance 를 보고한다.
  • Tab. 5 에서는 TriviaQA 에 대한 performance 를 보고한다.

두 benchmarks 모두에서, LLaMA-65B 는 zero-shot 및 few-shot settings 에서 state-of-the-art performance 를 달성한다. 더 중요한 점은, LLaMA-13B 역시 이 benchmarks 에서 GPT-3 및 Chinchilla 와 경쟁력이 있다는 점이며, 그럼에도 5–10× 더 작다. 이 model 은 inference 동안 single V100 GPU 에서 실행된다.

3.3 Reading Comprehension​

저자는 RACE reading comprehension benchmark 에서 저자의 models 를 평가한다. 이 dataset 은 중국의 middle school 및 high school 학생들을 위해 설계된 English reading comprehension 시험에서 수집되었다. 저자는 Brown et al. 의 evaluation setup 을 따르며, 결과를 Tab. 6 에 보고한다.

이 benchmarks 에서:

  • LLaMA-65B 는 PaLM-540B 와 경쟁력이 있다.
  • LLaMA-13B 는 GPT-3 를 몇 퍼센트 차이로 능가한다.

3.4 Mathematical reasoning​

저자는 두 개의 mathematical reasoning benchmarks 인 MATH 와 GSM8k 에서 저자의 models 를 평가한다.

  • MATH 는 LaTeX 로 작성된 12K 개의 middle school 및 high school mathematics problems 로 이루어진 dataset 이다.
  • GSM8k 는 middle school mathematical problems 의 집합이다.

Tab. 7 에서 저자는 PaLM 및 Minerva 와 비교한다.

  • Minerva 는 ArXiv 와 Math Web Pages 에서 추출된 38.5B tokens 에 대해 finetuning 된 PaLM models 의 시리즈인 반면, PaLM 과 LLaMA 는 모두 mathematical data 에 대해 finetuning 되지 않았다.
  • PaLM 과 Minerva 의 수치는 Lewkowycz et al. 에서 가져오며, 저자는 maj1@k 사용 여부 각각에 대해 비교한다.
  • maj1@k 는 각 problem 에 대해 kkk 개의 samples 를 생성하고 majority voting 을 수행하는 평가를 의미한다.
  • GSM8k 에서 저자는 LLaMA-65B 가 Minerva-62B 를 능가함을 관찰하며, 이는 mathematical data 에 대해 fine-tuning 되지 않았음에도 그렇다.

3.5 Code generation​

저자는 natural language description 으로부터 code 를 작성하는 저자의 models 의 능력을 두 개의 benchmarks 인 HumanEval 과 MBPP 에서 평가한다. 두 tasks 모두에서, model 은 몇 문장으로 이루어진 program description 과 몇 개의 input-output examples 를 받는다.

  • HumanEval 에서는 function signature 도 함께 제공되며, prompt 는 textual description 과 tests 를 docstring 에 포함한 natural code 형태로 formatting 된다.
  • model 은 description 에 부합하고 test cases 를 만족하는 Python program 을 생성해야 한다.

Tab. 8 에서 저자는 code 에 대해 finetuning 되지 않은 기존 language models 인 PaLM 과 LaMDA 와 저자의 model 의 pass@1 scores 를 비교한다. PaLM 과 LLaMA 는 유사한 수의 code tokens 를 포함하는 datasets 에서 training 되었다.

  • Tab. 8 에서 보이듯, 유사한 수의 parameters 에 대해 LLaMA 는 code 에 특화되어 training 또는 finetuning 되지 않은 다른 general models 인 LaMDA 와 PaLM 을 능가한다.
    • 13B parameters 이상을 가진 LLaMA 는 HumanEval 과 MBPP 모두에서 LaMDA 137B 를 능가한다.
    • LLaMA 65B 역시 더 오래 training 되었을 때조차 PaLM 62B 를 능가한다.
  • 이 table 에 보고된 pass@1 결과는 temperature 0.1 로 sampling 하여 얻었다.
    • pass@100 및 pass@80 metrics 는 temperature 0.8 로 얻었다.
    • 저자는 Chen et al. 과 동일한 method 를 사용하여 pass@k 의 unbiased estimates 를 얻는다.

code 에 대한 성능은 code-specific tokens 에 대해 finetuning 함으로써 향상시킬 수 있다.

  • 예를 들어, PaLM-Coder 는 HumanEval 에서 PaLM 의 pass@1 score 를 26.2% 에서 36% 로 증가시킨다.
  • code 에 특화되어 training 된 다른 models 역시 이러한 tasks 에서 general models 보다 더 나은 성능을 보인다.
  • code tokens 에 대한 finetuning 은 이 논문의 범위를 벗어난다.

3.6 Massive Multitask Language Understanding​

massive multitask language understanding benchmark, 또는 MMLU 는 Hendrycks et al. 이 도입한 benchmark 로, humanities, STEM, social sciences 를 포함한 다양한 knowledge domains 를 포괄하는 multiple choice questions 로 구성된다. 저자는 benchmark 에서 제공하는 examples 를 사용하여 5-shot setting 에서 저자의 models 를 평가하고, 결과를 Tab. 9 에 보고한다.

  • 이 benchmark 에서 저자는 LLaMA-65B 가 평균적으로, 그리고 대부분의 domains 전반에서 Chinchilla-70B 와 PaLM-540B 모두보다 몇 퍼센트 뒤처짐을 관찰한다.
  • 가능한 설명 중 하나는 저자가 pre-training data 에서 books 와 academic papers 를 제한된 양만 사용했다는 점이다.
  • 즉, ArXiv, Gutenberg, Books3 를 합쳐도 총 177GB 에 불과한 반면, 이들 models 는 최대 2TB 의 books 로 training 되었다.
  • Gopher, Chinchilla, PaLM 이 사용한 이 많은 양의 books 는 Gopher 가 다른 benchmarks 에서는 GPT-3 와 비슷하지만, 이 benchmark 에서는 GPT-3 를 능가하는 이유도 설명할 수 있다.

5 Bias, Toxicity and Misinformation

large language models 는 training data 에 존재하는 biases 를 재현하고 증폭시키는 것으로 알려져 있으며, toxic 하거나 offensive 한 content 를 생성하는 것으로도 알려져 있다. 저자의 training dataset 은 Web 에서 온 data 를 큰 비율로 포함하고 있으므로, 저자는 저자의 models 가 이러한 content 를 생성할 잠재력이 있는지를 판단하는 것이 중요하다고 믿는다.

LLaMA-65B 의 잠재적 harm 를 이해하기 위해, 저자는 toxic content production 과 stereotypes detection 을 측정하는 서로 다른 benchmarks 에서 평가를 수행한다. 저자가 language model community 에서 이러한 model 들의 문제를 나타내기 위해 사용되는 몇몇 표준 benchmarks 를 선택하였지만, 이러한 평가만으로는 이 models 와 관련된 risks 를 완전히 이해하기에 충분하지 않다.

5.1 RealToxicityPrompts​

language models 는 insults, hate speech, threats 와 같은 toxic language 를 생성할 수 있다. model 이 생성할 수 있는 toxic content 의 범위는 매우 넓어서, 이를 철저하게 평가하는 것은 어렵다. 최근 여러 연구는 RealToxicityPrompts benchmark 를 저자의 model 이 얼마나 toxic 한지를 나타내는 지표로 고려하였다. RealToxicityPrompts 는 약 100k 개의 prompts 로 구성되며, model 은 이를 completion 해야 한다. 그 다음 toxicity score 는 PerspectiveAPI 에 request 를 보내 자동으로 평가된다.

저자는 third-party 인 PerspectiveAPI 가 사용하는 pipeline 을 통제할 수 없기 때문에, 이전 models 와의 비교가 어렵다.

100k 개의 prompts 각각에 대해, 저자는 저자의 models 로 greedy generation 을 수행하고 toxicity score 를 측정한다. prompt 당 score 는 0 (non-toxic) 에서 1 (toxic) 까지의 범위를 가진다. Tab. 11 에서 저자는 RealToxicityPrompts 의 basic 및 respectful prompt categories 에 대한 평균 score 를 보고한다.

  • 이 score 들은 literature 에서 관찰되는 수치와 “비슷하다” (e.g., Chinchilla 의 경우 0.087).
  • 그러나 methodology 는 해당 연구들과 저자의 연구 사이에서 다르다. 예를 들어 sampling strategy, prompts 수, API 사용 시점이 다르다.
  • 저자는 toxicity 가 model 크기와 함께 증가함을 관찰하며, 특히 Respectful prompts 에서 그러하다.
    • 이는 이전 연구에서도 관찰되었다. 다만 Hoffmann et al. 에서는 크기가 다름에도 Chinchilla 와 Gopher 사이에 차이를 보지 않았다는 주목할 만한 예외가 있다.
    • 이는 더 큰 model 인 Gopher 가 Chinchilla 보다 더 나쁜 performance 를 가지기 때문일 수 있으며, toxicity 와 model size 의 관계는 model family 내부에서만 적용될 수 있음을 시사한다.

5.2 CrowS-Pairs​

저자는 CrowS-Pairs 에서 저자의 model 의 biases 를 평가한다. 이 dataset 은 gender, religion, race/color, sexual orientation, age, nationality, disability, physical appearance, socioeconomic status 의 9 개 categories 에 대한 bias 를 측정할 수 있게 한다. 각 example 은 stereotype 과 anti-stereotype 으로 구성되며, 저자는 zero-shot setting 에서 두 문장의 perplexity 를 사용하여 model 이 stereotypical sentence 를 선호하는 정도를 측정한다. 따라서 점수가 높을수록 bias 가 더 높음을 나타낸다.

저자는 Tab. 12 에서 GPT-3 및 OPT-175B 와 비교한다.

  • LLaMA 는 평균적으로 두 models 보다 약간 더 유리하게 비교된다. 저자의 model 은 특히 religion category 에서 편향이 크며 (OPT-175B 대비 +10%), 그 다음이 age 와 gender 이다.
  • 저자는 이러한 biases 가 여러 filtering steps 에도 불구하고 CommonCrawl 에서 비롯된다고 예상한다.

5.3 WinoGender​

gender category 에서 저자의 model 의 biases 를 더 자세히 조사하기 위해, 저자는 co-reference resolution dataset 인 WinoGender benchmark 를 살펴본다. WinoGender 는 Winograd schema 로 구성되며, pronoun 의 gender 가 model 의 co-reference resolution performance 에 영향을 미치는지를 통해 bias 를 평가한다.

  • 더 정확히 말하면, 각 sentence 는 세 개의 mentions 를 가진다: “occupation”, “participant”, 그리고 “pronoun” 이며, 여기서 pronoun 은 occupation 또는 participant 중 하나를 co-reference 한다.
  • 저자는 model 에게 co-reference relation 을 판단하도록 prompt 하며, sentence 의 context 에 따라 그것을 올바르게 수행하는지를 측정한다.
  • 목적은 occupations 과 연관된 societal biases 가 model 에 포착되었는지를 드러내는 것이다.
    • 예를 들어, WinoGender dataset 의 한 sentence 는 “The nurse notified the patient that his shift would be ending in an hour.” 이며, 그 뒤에 ‘His’ refers to 가 이어진다.
  • 이후 저자는 model 로 co-reference resolution 을 수행하기 위해 continuations 인 the nurse 와 the patient 의 perplexity 를 비교한다.
  • 저자는 3 개 pronouns 를 사용할 때의 performance 를 평가한다: “her/her/she”, “his/him/he”, “their/them/someone”. 여기서 각 선택은 pronoun 의 grammatical function 에 대응한다.

Tab. 13 에서 저자는 dataset 에 포함된 세 가지 서로 다른 pronouns 에 대한 co-reference scores 를 보고한다.

  • 저자는 저자의 model 이 “her/her/she” 와 “his/him/he” pronouns 보다 “their/them/someone” pronouns 에 대해 co-reference resolution 을 수행하는 데 훨씬 더 뛰어남을 관찰한다.
  • 유사한 관찰은 이전 연구에서도 이루어졌으며, 이는 gender bias 를 시사할 가능성이 높다.
    • 실제로 “her/her/she” 와 “his/him/he” pronouns 의 경우, model 은 sentence 의 evidence 를 사용하는 대신 occupation 의 majority gender 를 사용하여 co-reference resolution 을 수행하고 있을 가능성이 크다.

이 가설을 더 조사하기 위해, 저자는 WinoGender dataset 에서 “her/her/she” 와 “his/him/he” pronouns 에 대한 “gotcha” cases 집합을 살펴본다.

  • 이러한 cases 는 pronoun 이 occupation 의 majority gender 와 일치하지 않으면서도, occupation 이 정답인 sentences 에 해당한다.
  • Tab. 13 에서 저자는 저자의 model 인 LLaMA-65B 가 gotcha examples 에서 더 많은 errors 를 범함을 관찰하며, 이는 gender 와 occupation 과 관련된 societal biases 를 이 model 이 포착하고 있음을 명확히 보여준다.
  • performance 하락은 “her/her/she” 와 “his/him/he” pronouns 모두에서 존재하며, 이는 gender 와 무관하게 biases 가 있음을 시사한다.

5.4 TruthfulQA​

TruthfulQA 는 model 의 truthfulness, 즉 어떤 claim 이 true 인지를 식별하는 능력을 측정하는 것을 목표로 한다. Lin et al. 은 “true” 의 정의를 “현실 세계에 대한 literal truth” 의 의미로 고려하며, belief system 이나 tradition 의 맥락에서만 true 인 claims 는 포함하지 않는다. 이 benchmark 는 model 이 misinformation 이나 false claims 를 생성할 risks 를 평가할 수 있다. questions 는 다양한 style 로 작성되어 있고, 38 개 categories 를 다루며, adversarial 하도록 설계되었다.

Tab. 14 에서 저자는 truthful models 를 측정하기 위한 questions 과 truthful and informative 의 교집합 양쪽에 대해 저자의 models 의 performance 를 보고한다.

  • GPT-3 와 비교했을 때, 저자의 model 은 두 categories 모두에서 더 높은 점수를 기록하지만, correct answers 의 비율은 여전히 낮다. 이는 저자의 model 이 잘못된 답을 hallucinate 할 가능성이 높음을 보여준다.

6 Carbon footprint

저자의 models 의 training 은 막대한 양의 energy 를 소비하였고, 그 결과 carbon dioxide 배출이 발생하였다. 저자는 이 주제에 관한 최근 literature 를 따르며, 총 energy consumption 과 그로 인해 발생한 carbon footprint 를 Tab. 15 에서 각각 분석한다.

저자는 model training 에 필요한 Watt-hour, Wh 와 carbon emissions 의 tons 단위인 tCO2eqtCO2eqtCO2eq 를 추정하기 위해 Wu et al. 의 공식을 따른다. Wh 에 대해서 저자는 다음 공식을 사용한다:

Wh = GPU-h×(GPU power consumption)×PUE\text{Wh = GPU-h} \times \text{(GPU power consumption)} \times \text{PUE}Wh = GPU-h×(GPU power consumption)×PUE

여기서 저자는 Power Usage Effectiveness (PUE) 를 1.1 로 설정한다. 결과적인 carbon emission 은 network 를 training 하는 데 사용된 data center 의 위치에 따라 달라진다. 예를 들어, BLOOM 은 0.057,kg CO2eq/KWh0.057 , \text{kg } CO2eq/KWh0.057,kg CO2eq/KWh 를 배출하는 grid 를 사용하여 27,tCO2eq27 , tCO2eq27,tCO2eq 를 유발하고, OPT 는 0.231,kg CO2eq/KWh0.231 , \text{kg } CO2eq/KWh0.231,kg CO2eq/KWh 를 배출하는 grid 를 사용하여 82,tCO2eq82 , tCO2eq82,tCO2eq 를 유발한다. 이 연구에서 저자는 이들 models 가 동일한 data center 에서 training 되었다고 가정할 때 training 의 carbon emission cost 를 비교하는 데 관심이 있다. 따라서 저자는 data center 의 위치를 고려하지 않고, 대신 미국 전국 평균 carbon intensity factor 인 0.385,kg CO2eq/KWh0.385 , \text{kg } CO2eq/KWh0.385,kg CO2eq/KWh 를 사용한다. 이는 tons 단위 carbon emissions 에 대해 다음 공식을 이끈다:

tCO2eq = MWh×0.385\text{tCO2eq = MWh} \times 0.385tCO2eq = MWh×0.385

저자는 공정한 비교를 위해 동일한 공식을 OPT 와 BLOOM 에도 적용한다. OPT 에 대해서는, training 이 992 개의 A100-80B 에서 34 일 동안 필요했다고 가정한다.

마지막으로, 저자는 저자의 models 개발을 위해 약 5 개월 동안 2048 개의 A100-80GB 를 사용했다고 추정한다. 이는 저자의 가정하에서 이 models 의 개발이 약 2,638,MWh2{,}638 , MWh2,638,MWh 의 비용과 총 1,015,tCO2eq1{,}015 , tCO2eq1,015,tCO2eq 의 배출을 초래했음을 의미한다. 저자는 training 이 이미 완료되었고 일부 model 은 상대적으로 작아서 single GPU 에서 실행될 수 있기 때문에, 이 models 의 공개가 미래의 carbon emission 을 줄이는 데 도움이 되기를 기대한다.

7 Related work

language models 는 단어, tokens, 또는 characters 의 sequence 에 대한 probability distributions 이다. 이 task 는 흔히 next token prediction 으로 정식화되며, 오랫동안 natural language processing 의 핵심 문제로 여겨져 왔다. Turing 이 “imitation game” 을 통해 language 사용으로 machine intelligence 를 측정할 것을 제안했기 때문에, language modeling 은 artificial intelligence 를 향한 진전을 측정하는 benchmark 로 제안되어 왔다.

Architecture​

전통적으로, language models 는 n-gram count statistics 에 기반하였다. 희귀 사건의 추정을 개선하기 위해 다양한 smoothing techniques 이 제안되었다. 지난 20 년 동안, neural networks 는 language modelling task 에 성공적으로 적용되어 왔으며, feed forward models, recurrent neural networks, LSTMs 로 이어졌다. 보다 최근에는 self-attention 에 기반한 transformer networks 가 중요한 개선을 이끌어냈으며, 특히 long range dependencies 를 포착하는 데 강점을 보였다.

Scaling​

language models 에서 model 크기와 dataset 크기 모두에 대한 scaling 은 오랜 역사를 가진다.

  • Brants et al. 은 2 trillion tokens 로 training 된 language models, 즉 300 billion n-grams 를 사용하면 machine translation 품질에 이점이 있음을 보였다.
    • 이 연구는 Stupid Backoff 라는 단순한 smoothing technique 에 의존했지만,
    • 이후 Heafield et al. 은 Kneser-Ney smoothing 을 Web-scale data 로 scaling 하는 방법을 보였다.
  • 이는 CommonCrawl 의 975 billion tokens 에 대해 5-gram model 을 training 하는 것을 가능하게 했고, 그 결과 500 billion n-grams 를 가진 model 이 만들어졌다.
  • Chelba et al. 은 language models 의 진전을 측정하기 위한 large-scale training dataset 인 One Billion Word benchmark 를 도입하였다.
  • neural language models 의 맥락에서, Jozefowicz et al. 은 LSTMs 를 1 billion parameters 까지 scaling 함으로써 Billion Word benchmark 에서 state-of-the-art 결과를 얻었다.
  • 이후 transformers 의 scaling 은 많은 NLP tasks 에서 개선을 이끌었다.
    • 대표적인 models 로는 BERT, GPT-2, Megatron-LM, T5 가 있다.
    • 중요한 돌파구는 175 billion parameters 를 가진 GPT-3 에서 이루어졌다.
    • 이는 Jurassic-1, Megatron-Turing NLG, Gopher, Chinchilla, PaLM, OPT, GLM 과 같은 일련의 Large Language Models 로 이어졌다.
  • Hestness et al. 과 Rosenfeld et al. 은 scaling 이 deep learning models 의 performance 에 미치는 영향을 연구하였고, model 및 dataset 크기와 system performance 사이에 power laws 가 존재함을 보였다.
  • Kaplan et al. 은 transformer 기반 language models 에 대해 이러한 power laws 를 도출하였으며, 이후 Hoffmann et al. 이 dataset scaling 시 learning rate schedule 을 조정함으로써 이를 더 정교하게 다듬었다.
  • 마지막으로, Wei et al. 은 scaling 이 large language models 의 abilities 에 미치는 영향을 연구하였다.

8 Conclusion

이 논문에서 저자는 openly released 되었고 state-of-the-art foundation models 와 경쟁력 있는 일련의 language models 를 제시하였다. 특히, LLaMA-13B 는 GPT-3 보다 10× 이상 더 작으면서도 GPT-3 를 능가하며, LLaMA-65B 는 Chinchilla-70B 및 PaLM-540B 와 경쟁력이 있다. 이전 연구들과 달리, 저자는 proprietary datasets 에 의존하지 않고 오직 publicly available data 만으로 training 하여도 state-of-the-art performance 를 달성할 수 있음을 보였다. 저자는 이러한 models 를 research community 에 공개함으로써 large language models 의 발전이 가속화되고, robustness 를 개선하며 toxicity 와 bias 와 같은 알려진 문제를 완화하려는 노력을 돕기를 기대한다.

추가로, 저자는 Chung et al. 과 마찬가지로 instructions 에 대해 이러한 models 를 finetuning 하는 것이 유망한 결과로 이어짐을 관찰하였으며, future work 에서 이를 더 조사할 계획이다. 마지막으로, 저자는 scaling 에 따라 performance 가 지속적으로 향상되는 것을 확인하였기 때문에, 향후 더 큰 pretraining corpora 에서 training 된 더 큰 models 를 공개할 계획이다.
