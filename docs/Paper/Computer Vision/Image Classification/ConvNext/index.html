A ConvNet for the 2020s

논문 및 이미지 출처 : https://arxiv.org/pdf/2201.03545

Abstract

visual recognition 의 “Roaring 20s” 는 Vision Transformers (ViTs) 의 도입으로 시작되었으며, ViTs 는 image classification model 의 state-of-the-art 로서 빠르게 ConvNets 를 대체했다. 반면 vanilla ViT 는 object detection 및 semantic segmentation 과 같은 일반적인 computer vision task 에 적용될 때 어려움에 직면한다.

여러 ConvNet prior 를 다시 도입한 것은 hierarchical Transformers, 예를 들어 Swin Transformers 이며, 이는 Transformers 를 generic vision backbone 으로서 실용 가능하게 만들고 매우 다양한 vision task 에서 뛰어난 성능을 입증했다. 그러나 이러한 hybrid approach 의 효과는 여전히 convolutions 의 고유한 inductive bias 보다는 Transformers 의 본질적인 우월성 덕분으로 크게 여겨진다.

이 연구에서 저자는 design space 를 다시 검토하고 pure ConvNet 이 달성할 수 있는 한계를 시험한다.

  • 저자는 standard ResNet 을 vision Transformer 의 설계 방향으로 점진적으로 “modernize” 하며, 그 과정에서 성능 차이에 기여하는 몇 가지 핵심 component 를 발견한다.
  • 이 탐구의 결과는 ConvNeXt 라고 명명된 pure ConvNet model family 이다.
  • 전적으로 standard ConvNet module 로 구성된 ConvNeXt 는 accuracy 와 scalability 측면에서 Transformers 와 유리하게 경쟁하며, 87.8% 의 ImageNet top-1 accuracy 를 달성하고 COCO detection 및 ADE20K segmentation 에서 Swin Transformers 를 능가하는 동시에, standard ConvNets 의 단순성과 efficiency 를 유지한다.

1. Introduction

2010 년대를 돌아보면, 이 10 년은 deep learning 의 거대한 진보와 영향으로 특징지어졌다. 주요한 동력은 neural networks, 특히 convolutional neural networks (ConvNets) 의 부흥이었다. 이 10 년 동안 visual recognition 분야는 feature engineering 에서 (ConvNet) architecture 설계로 성공적으로 전환했다. back-propagation 으로 학습되는 ConvNets 의 발명은 1980 년대까지 거슬러 올라가지만, visual feature learning 에 대한 진정한 잠재력이 드러난 것은 2012 년 말이 되어서였다.

  • AlexNet 의 도입은 “ImageNet moment” 를 촉발했고, computer vision 의 새로운 시대를 열었다.
  • 이후 이 분야는 빠른 속도로 발전해 왔다. VGGNet, Inceptions, ResNe(X)t, DenseNet, MobileNet, EfficientNet, RegNet 과 같은 대표적인 ConvNets 는 accuracy, efficiency, scalability 의 서로 다른 측면에 초점을 맞추었고, 많은 유용한 design principle 을 대중화했다.

computer vision 에서 ConvNets 가 완전히 지배적이었던 것은 우연이 아니었다. 많은 application scenario 에서 “sliding window” 전략은 visual processing 에 본질적이며, 특히 high-resolution image 를 다룰 때 그렇다.

  • ConvNets 는 다양한 computer vision application 에 잘 맞도록 하는 여러 built-in inductive bias 를 가진다.
  • 그중 가장 중요한 것은 translation equivariance 이며, 이는 object detection 과 같은 task 에 바람직한 특성이다.
  • 또한 ConvNets 는 sliding-window 방식으로 사용될 때 computation 이 공유된다는 점 때문에 본질적으로 efficient 하다.
  • 수십 년 동안 이는 ConvNets 의 기본 사용 방식이었고, 일반적으로 digit, face, pedestrian 과 같은 제한된 object category 에 적용되었다.
  • 2010 년대에 들어서 region-based detector 는 ConvNets 를 visual recognition system 의 근본적인 building block 위치로 더욱 끌어올렸다.

비슷한 시기에 natural language processing (NLP) 를 위한 neural network 설계의 여정은 매우 다른 길을 걸었고, Transformers 가 recurrent neural networks 를 대체하여 지배적인 backbone architecture 가 되었다. language 와 vision domain 사이에 관심 task 의 차이가 있었음에도, 두 흐름은 놀랍게도 2020 년에 수렴했는데, Vision Transformers (ViT) 의 도입이 network architecture 설계의 지형을 완전히 바꾸었기 때문이다.

  • image 를 patch sequence 로 분할하는 초기 “patchify” layer 를 제외하면, ViT 는 image-specific inductive bias 를 도입하지 않으며 원래의 NLP Transformers 에 최소한의 변경만 가한다.
  • ViT 의 주요 초점 중 하나는 scaling behavior 이다. larger model 과 dataset size 의 도움으로, Transformers 는 standard ResNets 를 큰 폭으로 능가할 수 있다.
  • image classification task 에서의 이러한 결과는 고무적이지만, computer vision 은 image classification 에 국한되지 않는다.

앞서 논의했듯이, 지난 10 년간 수많은 computer vision task 에 대한 해법은 sliding-window, fully-convolutional paradigm 에 크게 의존했다. ConvNet inductive bias 가 없으면, vanilla ViT model 은 generic vision backbone 으로 채택되는 데 많은 어려움에 직면한다. 가장 큰 문제는 ViT 의 global attention 설계이며, 이는 input size 에 대해 quadratic complexity 를 갖는다. 이는 ImageNet classification 에서는 허용 가능할 수 있지만, 더 높은 resolution input 에서는 빠르게 감당 불가능해진다.

Hierarchical Transformers 는 이러한 간극을 메우기 위해 hybrid approach 를 사용한다.

  • 예를 들어, “sliding window” 전략, 즉 local window 내 attention 이 다시 Transformers 에 도입되었고, 이를 통해 Transformers 가 ConvNets 와 더 유사하게 동작할 수 있게 되었다.
  • Swin Transformer 는 이 방향의 이정표가 되는 연구로, image classification 을 넘어선 다양한 computer vision task 에서 Transformers 가 generic vision backbone 으로 채택될 수 있고 state-of-the-art 성능을 달성할 수 있음을 처음으로 보여주었다.
  • Swin Transformer 의 성공과 빠른 채택은 한 가지 사실도 드러냈다. convolution 의 본질은 중요하지 않게 된 것이 아니라, 여전히 매우 필요하며 결코 사라지지 않았다는 점이다.

이러한 관점에서 보면, computer vision 을 위한 Transformers 의 많은 발전은 convolution 을 다시 가져오는 데 목적을 두고 있었다. 그러나 이러한 시도에는 비용이 따른다.

  • sliding window self-attention 의 naive implementation 은 비쌀 수 있으며, cyclic shifting 과 같은 발전된 접근을 사용하면 속도는 최적화할 수 있지만 system 설계는 더 정교해진다.
  • 반면 거의 아이러니하게도, ConvNet 은 이미 그러한 바람직한 특성의 많은 부분을, 비록 단순하고 꾸밈없는 방식이지만, 만족한다.
  • ConvNets 가 추진력을 잃는 것처럼 보이는 유일한 이유는 (hierarchical) Transformers 가 많은 vision task 에서 이를 능가하기 때문이며, 그 성능 차이는 보통 Transformers 의 우수한 scaling behavior 때문으로 여겨지고, multi-head self-attention 이 핵심 component 로 간주된다.

지난 10 년 동안 점진적으로 개선되어 온 ConvNets 와 달리, Vision Transformers 의 채택은 단계적 변화였다.

  • 최근 문헌에서는 두 가지를 비교할 때 보통 system-level 비교, 예를 들어 Swin Transformer 대 ResNet 이 사용된다.
  • ConvNets 와 hierarchical vision Transformers 는 동시에 다르고 동시에 비슷해진다.
  • 둘 다 유사한 inductive bias 를 갖추고 있지만, training procedure 와 macro/micro-level architecture 설계에서는 크게 다르다.

이 연구에서 저자는 ConvNets 와 Transformers 사이의 architectural distinction 을 조사하고, network performance 를 비교할 때의 confounding variable 을 식별하려고 한다. 저자의 연구는 ConvNets 에 대해 pre-ViT 시대와 post-ViT 시대 사이의 간극을 메우는 동시에, pure ConvNet 이 달성할 수 있는 한계를 시험하려는 목적을 가진다.

이를 위해 저자는 개선된 procedure 로 학습된 standard ResNet, 예를 들어 ResNet-50 에서 시작한다.

  • 저자는 architecture 를 hierarchical vision Transformer, 예를 들어 Swin-T 의 구성 방향으로 점진적으로 “modernize” 한다.
  • 저자의 탐구는 하나의 핵심 질문에 의해 이끌린다. Transformer 의 design decision 이 ConvNets 의 성능에 어떤 영향을 미치는가? 저자는 그 과정에서 성능 차이에 기여하는 몇 가지 핵심 component 를 발견한다.
  • 그 결과 저자는 ConvNeXt 라고 부르는 pure ConvNets family 를 제안한다.
  • 저자는 ConvNeXts 를 ImageNet classification, COCO 에서의 object detection/segmentation, ADE20K 에서의 semantic segmentation 과 같은 다양한 vision task 에서 평가한다.
  • 놀랍게도, 전적으로 standard ConvNet module 로 구성된 ConvNeXts 는 accuracy, scalability, robustness 측면에서 모든 주요 benchmark 에서 Transformers 와 유리하게 경쟁한다.
  • ConvNeXt 는 standard ConvNets 의 efficiency 를 유지하며, training 과 testing 모두에서 fully-convolutional 한 특성 덕분에 구현이 매우 단순하다.

저자는 이러한 새로운 관찰과 논의가 몇몇 일반적인 믿음에 도전하고, computer vision 에서 convolution 의 중요성을 다시 생각하도록 장려하기를 바란다.

2. Modernizing a ConvNet: a Roadmap

이 section 에서 저자는 ResNet 에서 시작하여 Transformers 와 유사성을 가지는 ConvNet 으로 나아가는 trajectory 를 제시한다. 저자는 FLOPs 기준으로 두 가지 model size 를 고려한다. 하나는 약 4.5×1094.5 \times 10^94.5×109 FLOPs 를 갖는 ResNet-50 / Swin-T regime 이고, 다른 하나는 약 15.0×10915.0 \times 10^915.0×109 FLOPs 를 갖는 ResNet-200 / Swin-B regime 이다. 단순화를 위해 저자는 ResNet-50 / Swin-T complexity model 의 result 를 제시한다. 더 높은 capacity model 에 대해서도 결론은 일관되며, result 는 Appendix C 에서 찾을 수 있다.

높은 수준에서 보면, 저자의 탐구는 standard ConvNet 으로서 network 의 단순성을 유지하면서 Swin Transformer 의 서로 다른 수준의 design 을 조사하고 따르는 방향으로 이루어진다. 저자의 탐구 roadmap 은 다음과 같다. 출발점은 ResNet-50 model 이다.

저자는 먼저 vision Transformers 를 학습시키는 데 사용된 것과 유사한 training technique 으로 이 model 을 학습시키고, 원래의 ResNet-50 과 비교해 훨씬 향상된 result 를 얻는다. 이것이 baseline 이 된다. 이후 저자는 일련의 design decision 을 연구하며, 이를 1) macro design, 2) ResNeXt, 3) inverted bottleneck, 4) large kernel size, 5) 다양한 layer-wise micro design 으로 요약한다. Fig. 2 에서 저자는 “network modernization” 의 각 step 에서 수행한 procedure 와 달성 가능한 result 를 보여준다.

network complexity 는 최종 성능과 밀접하게 상관되어 있으므로, 탐구 전반에 걸쳐 FLOPs 는 대체로 제어된다. 다만 중간 step 에서는 FLOPs 가 reference model 보다 높거나 낮을 수 있다. 모든 model 은 ImageNet-1K 에서 학습되고 평가된다.

2.1. Training Techniques​

network architecture 의 design 외에도 training procedure 역시 최종 성능에 영향을 준다. vision Transformers 는 새로운 module 과 architectural design decision 집합을 가져왔을 뿐 아니라, vision 분야에 서로 다른 training technique, 예를 들어 AdamW optimizer 도 도입했다. 이는 주로 optimization strategy 와 이에 연관된 hyper-parameter setting 과 관련된다. 따라서 저자의 탐구 첫 단계는 vision Transformer training procedure 로 baseline model, 이 경우 ResNet-50/200 을 학습시키는 것이다. 최근 연구들은 modern training technique 집합이 단순한 ResNet-50 model 의 성능을 크게 향상시킬 수 있음을 보여준다.

저자의 연구에서는 DeiT 와 Swin Transformer 의 recipe 에 가까운 training recipe 를 사용한다.

  • 학습은 ResNets 의 원래 90 epochs 에서 300 epochs 로 연장된다.
  • 저자는 AdamW optimizer 를 사용하고, Mixup, Cutmix, RandAugment, Random Erasing 과 같은 data augmentation technique, 그리고 Stochastic Depth 와 Label Smoothing 을 포함한 regularization scheme 을 사용한다.
  • 사용한 전체 hyper-parameter 집합은 Appendix A.1 에서 찾을 수 있다.

이 향상된 training recipe 만으로도 ResNet-50 model 의 성능은 76.1% 에서 78.8% 로 증가했으며, 이는 +2.7+2.7%+2.7 에 해당한다. 이는 전통적인 ConvNets 와 vision Transformers 사이 성능 차이의 상당 부분이 training technique 때문일 수 있음을 시사한다.

저자는 “modernization” process 전체에서 같은 hyper-parameter 를 사용한 이 고정된 training recipe 를 사용할 것이다. ResNet-50 regime 에서 보고된 각 accuracy 는 서로 다른 세 random seed 로 학습한 결과의 평균값이다.

2.2. Macro Design​

이제 저자는 Swin Transformers 의 macro network design 을 분석한다.

Swin Transformers 는 feature map resolution 이 서로 다른 각 stage 로 이루어진 multi-stage design 을 사용하기 위해 ConvNets 를 따른다. 여기에는 두 가지 흥미로운 design 고려 사항이 있다.

stage compute ratio 와 “stem cell” structure 이다.

Changing stage compute ratio.​

ResNet 에서 stage 전반에 걸친 computation distribution 의 원래 design 은 대체로 경험적으로 결정되었다.

  • 무거운 “res4” stage 는 detector head 가 14×1414 \times 1414×14 feature plane 에서 작동하는 object detection 과 같은 downstream task 와 호환되도록 의도되었다.
  • 반면 Swin-T 는 같은 원칙을 따르지만, stage compute ratio 는 약간 다른 1:1:3:11:1:3:11:1:3:1 이다.
  • 더 큰 Swin Transformers 에서는 ratio 가 1:1:9:11:1:9:11:1:9:1 이다.
    • 이 design 을 따라 저자는 각 stage 의 block 수를 ResNet-50 의 (3,4,6,3)(3, 4, 6, 3)(3,4,6,3) 에서 (3,3,9,3)(3, 3, 9, 3)(3,3,9,3) 으로 조정하며, 이는 FLOPs 를 Swin-T 와도 정렬시킨다.
    • 이로 인해 model accuracy 는 78.8% 에서 79.4% 로 향상된다.
  • 주목할 점은, 연구자들이 computation distribution 을 철저히 조사해 왔으며, 더 최적인 design 이 존재할 가능성이 있다는 것이다.

이후부터 저자는 이 stage compute ratio 를 사용한다.

Changing stem to “Patchify”.​

일반적으로 stem cell design 은 input image 를 network 시작 부분에서 어떻게 처리할 것인지와 관련된다. natural image 에 내재한 redundancy 때문에, standard ConvNets 와 vision Transformers 모두에서 일반적인 stem cell 은 input image 를 적절한 feature map size 로 적극적으로 downsample 한다.

  • standard ResNet 의 stem cell 은 stride 2 를 가진 7×77 \times 77×7 convolution layer 와 그 뒤의 max pool 로 구성되며, 그 결과 input image 는 4 배 downsampling 된다.
  • vision Transformers 에서는 더 공격적인 “patchify” 전략이 stem cell 로 사용되며, 이는 큰 kernel size, 예를 들어 kernel size =14= 14=14 또는 161616 과 non-overlapping convolution 에 대응한다.
  • Swin Transformer 는 architecture 의 multi-stage design 을 수용하기 위해 더 작은 patch size 4 를 사용하지만, 유사한 “patchify” layer 를 사용한다.
  • 저자는 ResNet-style stem cell 을 4×44 \times 44×4, stride 4 convolutional layer 로 구현된 patchify layer 로 대체한다.
  • accuracy 는 79.4% 에서 79.5% 로 변한다. 이는 ResNet 의 stem cell 이 ViT 스타일의 더 단순한 “patchify” layer 로 대체될 수 있으며, 그 결과 유사한 성능이 나온다는 것을 시사한다.
  • 저자는 network 에서 “patchify stem” (4×44 \times 44×4 non-overlapping convolution) 을 사용할 것이다.

2.3. ResNeXt-ify​

이 part 에서 저자는 vanilla ResNet 보다 더 나은 FLOPs/accuracy trade-off 를 가진 ResNeXt 의 아이디어를 도입하려고 시도한다. 핵심 component 는 grouped convolution 이며, convolutional filter 가 서로 다른 group 으로 분리된다. 높은 수준에서 ResNeXt 의 guiding principle 은 “use more groups, expand width” 이다. 보다 정확히 말하면, ResNeXt 는 bottleneck block 의 3×33 \times 33×3 conv layer 에 grouped convolution 을 사용한다. 이는 FLOPs 를 크게 줄이므로, network width 를 확장하여 capacity 손실을 보상한다.

저자의 경우에는 grouped convolution 의 특수한 경우인 depthwise convolution 을 사용한다.

  • 여기서 group 수는 channel 수와 같다. Depthwise conv 는 MobileNet 과 Xception 에 의해 널리 알려졌다.
  • 저자는 depthwise convolution 이 self-attention 의 weighted sum operation 과 유사하다고 언급하는데, 이는 per-channel basis 로 작동하며, 즉 spatial dimension 에서만 정보를 mixing 한다.
  • depthwise conv 와 1×11 \times 11×1 conv 의 조합은 spatial mixing 과 channel mixing 의 분리를 이끌어내며, 이는 vision Transformers 와 공유되는 특성이다.
  • vision Transformers 에서는 각 operation 이 spatial 또는 channel dimension 중 하나에서만 정보를 mixing 하고, 둘 다 동시에 하지는 않는다.
  • depthwise convolution 의 사용은 예상대로 network FLOPs 와 accuracy 를 함께 감소시킨다.
  • ResNeXt 에서 제안된 전략에 따라 저자는 network width 를 Swin-T 와 같은 channel 수인 96 으로 증가시킨다.
  • 기존 값은 64 였다. 이는 증가된 FLOPs (5.3G)(5.3\text{G})(5.3G) 와 함께 network performance 를 80.5% 까지 끌어올린다.

저자는 이제 ResNeXt design 을 사용할 것이다.

2.4. Inverted Bottleneck​

모든 Transformer block 의 중요한 design 중 하나는 inverted bottleneck 을 만든다는 점이다.

  • 즉, MLP block 의 hidden dimension 이 input dimension 보다 4 배 더 넓다. Sec. 2 에 이어 설명된 Fig. 4 를 참조할 수 있다.
  • 흥미롭게도, 이러한 Transformer design 은 ConvNets 에서 expansion ratio 4 로 사용되는 inverted bottleneck design 과 연결된다.
  • 이 아이디어는 MobileNetV2 에 의해 널리 알려졌고, 이후 여러 advanced ConvNet architecture 에서 주목받았다.

여기서 저자는 inverted bottleneck design 을 탐구한다.

  • Fig. 3 의 (a) 에서 (b) 는 configuration 을 보여준다. depthwise convolution layer 의 FLOPs 가 증가함에도 불구하고, 이 변경은 전체 network FLOPs 를 4.6G 로 줄인다.
    • 이는 downsampling residual block 의 shortcut 1×11 \times 11×1 conv layer 에서 FLOPs 가 크게 줄어들기 때문이다.
    • 흥미롭게도, 이는 성능을 소폭 향상시키며 80.5% 에서 80.6% 로 증가한다.
  • ResNet-200 / Swin-B regime 에서는 이 step 이 더 큰 gain 을 가져오며, FLOPs 도 줄어든 상태에서 81.9% 에서 82.6% 로 향상된다.

저자는 이제 inverted bottleneck 을 사용할 것이다.

2.5. Large Kernel Sizes​

이 part 의 탐구에서 저자는 large convolutional kernel 의 behavior 에 초점을 맞춘다.

  • vision Transformers 의 가장 두드러진 특징 중 하나는 non-local self-attention 이며, 이는 각 layer 가 global receptive field 를 갖도록 한다.
  • ConvNets 에서도 과거에 large kernel size 가 사용된 적은 있지만, gold standard 는 작은 kernel size 인 (3×3)(3 \times 3)(3×3) conv layer 를 쌓는 것이며, 이는 modern GPUs 에서 efficient hardware implementation 을 가진다.
  • Swin Transformers 가 self-attention block 에 local window 를 다시 도입했음에도, window size 는 최소 7×77 \times 77×7 이며, 이는 ResNe(X)t 의 3×33 \times 33×3 kernel size 보다 상당히 크다.

여기서 저자는 ConvNets 를 위해 large kernel-sized convolution 의 사용을 다시 검토한다.

Moving up depthwise conv layer.​

large kernel 을 탐구하기 위한 선행 조건 중 하나는 depthwise conv layer 의 위치를 위로 이동시키는 것이다. 이는 Fig. 3 의 (b) 에서 (c) 로 표현된다. 이것 역시 Transformers 에서 드러나는 design decision 이다.

MSA block 이 MLP layer 보다 앞에 놓인다. 저자는 inverted bottleneck block 을 가지고 있으므로, 이는 자연스러운 design choice 이다. 복잡하거나 비효율적인 module 인 MSA 와 large-kernel conv 는 더 적은 channel 을 가지게 되고, efficient 하고 dense 한 1×11 \times 11×1 layer 가 주요 연산을 담당한다. 이 중간 단계는 FLOPs 를 4.1G 로 줄이지만, 성능은 일시적으로 79.9% 로 저하된다.

Increasing the kernel size.​

이러한 준비가 모두 끝난 뒤에는, 더 큰 kernel-sized convolution 을 채택하는 이점이 뚜렷하다. 저자는 3, 5, 7, 9, 11 을 포함한 여러 kernel size 를 실험했다.

  • network 의 성능은 79.9% (3×3)(3 \times 3)(3×3) 에서 80.6% (7×7)(7 \times 7)(7×7) 로 증가하며, 이때 network 의 FLOPs 는 대체로 동일하게 유지된다.
  • 또한 저자는 더 큰 kernel size 의 이점이 7×77 \times 77×7 에서 saturation point 에 도달함을 관찰한다.
  • 저자는 이러한 behavior 를 large capacity model 에서도 검증했으며, ResNet-200 regime model 은 kernel size 를 7×77 \times 77×7 보다 더 크게 늘려도 추가 gain 을 보이지 않는다.

저자는 각 block 에서 7×77 \times 77×7 depthwise conv 를 사용할 것이다.

이 시점에서 저자는 macro scale 에서의 network architecture 검토를 마무리했다. 흥미롭게도, vision Transformer 에서 취해진 design choice 의 상당 부분이 ConvNet instantiation 으로 대응될 수 있다.

2.6. Micro Design​

이 section 에서 저자는 micro scale 에서의 여러 architectural difference 를 조사한다. 여기서의 대부분의 탐구는 layer level 에서 수행되며, activation function 과 normalization layer 의 구체적 선택에 초점을 맞춘다.

Replacing ReLU with GELU​

NLP 와 vision architecture 사이의 한 가지 차이는 어떤 activation function 을 사용할지에 관한 세부 사항이다. 시간이 지나며 수많은 activation function 이 개발되었지만, Rectified Linear Unit (ReLU) 는 그 단순성과 efficiency 때문에 여전히 ConvNets 에서 광범위하게 사용된다. ReLU 는 원래 Transformer 논문에서도 activation function 으로 사용된다.

Gaussian Error Linear Unit, 즉 GELU 는 ReLU 의 더 smooth 한 variant 로 볼 수 있으며, Google 의 BERT 와 OpenAI 의 GPT-2, 그리고 가장 최근에는 ViTs 를 포함한 가장 advanced 한 Transformers 에서 사용된다. 저자는 ConvNet 에서도 ReLU 를 GELU 로 대체할 수 있음을 확인했지만, accuracy 는 변하지 않고 80.6% 로 유지된다.

Fewer activation functions.​

Transformer 와 ResNet block 사이의 작은 차이 중 하나는 Transformer 에 activation function 이 더 적다는 점이다.

key/query/value linear embedding layer, projection layer, 그리고 MLP block 의 두 linear layer 로 이루어진 Transformer block 을 생각해 보자. activation function 은 MLP block 에 단 하나만 존재한다. 반면 ResNet 에서는 1×11 \times 11×1 conv 를 포함한 각 convolutional layer 뒤에 activation function 을 붙이는 것이 일반적이다. 여기서 저자는 같은 전략을 따를 때 성능이 어떻게 변하는지 조사한다.

  • Fig. 4 에 나타난 것처럼, 저자는 residual block 에서 두 개의 1×11 \times 11×1 layer 사이에 있는 GELU 하나만 남기고 나머지 모든 GELU layer 를 제거하여 Transformer block 의 style 을 재현한다.
  • 이 과정은 result 를 0.7% 향상시켜 81.3% 에 이르게 하며, 사실상 Swin-T 의 성능과 맞먹는다.

저자는 이제 각 block 에서 단 하나의 GELU activation 을 사용할 것이다.

Fewer normalization layers.​

Transformer block 은 보통 normalization layer 수도 더 적다. 여기서 저자는 두 개의 BatchNorm (BN) layer 를 제거하고, conv 1×11 \times 11×1 layer 앞에 단 하나의 BN layer 만 남긴다. 이는 성능을 81.4% 로 더 끌어올리며, 이미 Swin-T 의 result 를 능가한다. 저자는 경험적으로 block 시작 부분에 추가 BN layer 하나를 더 넣어도 성능이 향상되지 않음을 발견했기 때문에, block 당 normalization layer 수는 Transformers 보다도 더 적다.

Substituting BN with LN.​

BatchNorm 은 convergence 를 개선하고 overfitting 을 줄이기 때문에 ConvNets 에서 필수적인 component 이다. 그러나 BN 은 model 성능에 해로운 영향을 줄 수 있는 많은 복잡성을 함께 가진다. alternative normalization technique 을 개발하려는 수많은 시도가 있었지만, 대부분의 vision task 에서 BN 은 여전히 선호되는 선택지로 남아 있다. 반면 더 단순한 Layer Normalization (LN) 은 Transformers 에서 사용되어 서로 다른 application scenario 에 걸쳐 좋은 성능을 보여주었다.

원래 ResNet 에서 LN 으로 BN 을 직접 대체하면 suboptimal 한 성능이 나온다. network architecture 와 training technique 에 대한 모든 수정 이후, 저자는 여기서 BN 대신 LN 을 사용하는 영향력을 다시 검토한다. 저자는 저자의 ConvNet model 이 LN 으로도 학습에 어려움이 없음을 관찰하며, 실제로 성능은 약간 더 좋아져 81.5% 의 accuracy 를 얻는다.

이후부터 저자는 각 residual block 에서 normalization choice 로 하나의 LayerNorm 을 사용할 것이다.

Separate downsampling layers.​

ResNet 에서는 spatial downsampling 이 각 stage 시작의 residual block 에 의해 달성되며, stride 2 를 가진 3×33 \times 33×3 conv 와 shortcut connection 에서 stride 2 를 가진 1×11 \times 11×1 conv 를 사용한다. Swin Transformers 에서는 stage 사이에 별도의 downsampling layer 가 추가된다.

저자는 유사한 전략을 탐구하며, spatial downsampling 을 위해 stride 2 를 가진 2×22 \times 22×2 conv layer 를 사용한다. 이 수정은 놀랍게도 training divergence 로 이어진다. 추가 조사 결과, spatial resolution 이 바뀌는 모든 곳에 normalization layer 를 추가하면 training 을 안정화하는 데 도움이 됨을 보여준다.

여기에는 Swin Transformers 에서도 사용되는 여러 LN layer 가 포함된다.

  • 각 downsampling layer 앞에 하나
  • stem 뒤에 하나
  • 마지막 global average pooling 뒤에 하나

이를 통해 저자는 accuracy 를 82.0% 까지 향상시킬 수 있으며, 이는 Swin-T 의 81.3% 를 크게 초과한다.

저자는 separate downsampling layer 를 사용할 것이다. 이것으로 저자가 ConvNeXt 라고 이름 붙인 최종 model 에 도달한다. ResNet, Swin, ConvNeXt block structure 의 비교는 Fig. 4 에서 찾을 수 있다. ResNet-50, Swin-T, ConvNeXt-T 의 자세한 architecture specification 비교는 Tab. 9 에서 찾을 수 있다.

Closing remarks.​

저자는 첫 번째 “playthrough” 를 마쳤고, 이 compute regime 에서 ImageNet-1K classification 에 대해 Swin Transformer 를 능가할 수 있는 pure ConvNet 인 ConvNeXt 를 발견했다. 주목할 점은, 지금까지 논의된 모든 design choice 가 vision Transformers 로부터 적응되었다는 것이다. 또한 이러한 design 은 ConvNet 문헌 내에서도 새로운 것이 아니며, 지난 10 년 동안 모두 각각 따로 연구되었지만 집합적으로 연구되지는 않았다.

저자의 ConvNeXt model 은 Swin Transformer 와 대략 동일한 FLOPs, #params., throughput, memory use 를 가지지만, shifted window attention 이나 relative position bias 와 같은 specialized module 을 필요로 하지 않는다.

이러한 발견은 고무적이지만 아직 완전히 설득력 있지는 않다. 지금까지의 탐구는 작은 scale 에 제한되어 있었지만, vision Transformers 를 진정으로 구별하는 것은 scaling behavior 이기 때문이다. 또한 ConvNet 이 object detection 과 semantic segmentation 과 같은 downstream task 에서 Swin Transformers 와 경쟁할 수 있는지의 문제는 computer vision practitioner 에게 핵심적인 관심사이다.

다음 section 에서 저자는 data 와 model size 양쪽 측면에서 ConvNeXt model 을 scale up 하고, 다양한 visual recognition task 집합에서 이를 평가할 것이다.

3.1. Settings​

ImageNet-1K dataset 는 1000 개의 object class 와 1.2M 개의 training image 로 구성된다.

저자는 validation set 에서의 ImageNet-1K top-1 accuracy 를 보고한다. 또한 저자는 pre-training 을 위해 약 14M 개의 image 를 가진, 21841 개 class 의 더 큰 dataset 인 ImageNet-22K 에서 pre-training 을 수행하고, 이후 평가를 위해 pre-trained model 을 ImageNet-1K 에서 fine-tune 한다. 저자는 training setup 을 아래와 같이 요약한다. 더 자세한 내용은 Appendix A 에서 찾을 수 있다.

Training on ImageNet-1K.​

  • 저자는 learning rate 4e−34e-34e−3 을 사용하여 AdamW 로 ConvNeXts 를 300 epochs 동안 학습한다.
  • 20-epoch linear warmup 이 있고, 그 이후에는 cosine decaying schedule 을 사용한다.
  • 저자는 batch size 4096 과 weight decay 0.05 를 사용한다. data augmentation 으로는 Mixup, Cutmix, RandAugment, Random Erasing 을 포함한 일반적인 scheme 을 채택한다.
  • network regularization 에는 Stochastic Depth 와 Label Smoothing 을 사용한다.
  • 초기값 1e−61e-61e−6 의 Layer Scale 이 적용된다. 저자는 larger model 의 overfitting 을 완화한다는 점을 발견했기 때문에 Exponential Moving Average (EMA) 를 사용한다.

Pre-training on ImageNet-22K.​

저자는 5 epochs warmup 과 함께 ConvNeXts 를 ImageNet-22K 에서 90 epochs 동안 pre-train 한다. EMA 는 사용하지 않는다. 다른 setting 은 ImageNet-1K 를 따른다.

Fine-tuning on ImageNet-1K.​

  • 저자는 ImageNet-22K pre-trained model 을 ImageNet-1K 에서 30 epochs 동안 fine-tune 한다.
  • 저자는 AdamW, learning rate 5e−55e-55e−5, cosine learning rate schedule, layer-wise learning rate decay, warmup 없음, batch size 512, weight decay 1e−81e-81e−8 을 사용한다.
  • 기본 pre-training, fine-tuning, testing resolution 은 2242224^22242 이다.
  • 추가로 저자는 ImageNet-22K 및 ImageNet-1K pre-trained model 모두에 대해 더 큰 resolution 인 3842384^23842 에서도 fine-tune 한다.
  • ViTs/Swin Transformers 와 비교할 때, ConvNeXts 는 network 가 fully-convolutional 이고 input patch size 를 조정하거나 absolute/relative position bias 를 interpolate 할 필요가 없기 때문에, 서로 다른 resolution 에서 fine-tune 하기가 더 단순하다.

3.2. Results​

ImageNet-1K.​

Tab. 1 (upper) 은 두 개의 최근 Transformer variant 인 DeiT 와 Swin Transformers, 그리고 architecture search 로부터 나온 두 ConvNets 인 RegNets, EfficientNets, EfficientNetsV2 와의 result 비교를 보여준다.

  • ConvNeXt 는 accuracy-computation trade-off 와 inference throughput 측면에서 두 개의 강력한 ConvNet baseline 인 RegNet 과 EfficientNet 과 유리하게 경쟁한다.
  • ConvNeXt 는 유사한 complexity 의 Swin Transformer 를 전반적으로 능가하며, 때로는 상당한 margin 으로 앞선다. 예를 들어 ConvNeXt-T 에서는 0.8% 이다.
  • shifted window 나 relative position bias 와 같은 specialized module 없이도, ConvNeXts 는 Swin Transformers 와 비교해 향상된 throughput 을 가진다.

result 에서 두드러지는 사례는 3842384^23842 의 ConvNeXt-B 이다.

  • 이는 Swin-B 를 0.6% 차이로 능가한다. 즉, 85.1% 대 84.5% 이다.
  • 동시에 inference throughput 은 12.5% 더 높다. 즉, 95.7 image/s 대 85.1 image/s 이다.
  • 저자는 resolution 이 2242224^22242 에서 3842384^23842 로 증가할수록 ConvNeXt-B 가 Swin-B 에 대해 가지는 FLOPs/throughput 이점이 더 커진다는 점을 언급한다.
  • 또한 ConvNeXt-L 로 further scaling 할 경우 85.5% 의 향상된 result 가 관찰된다.

ImageNet-22K.​

저자는 Tab. 1 (lower) 에서 ImageNet-22K pre-training 으로부터 fine-tune 된 model 의 result 를 제시한다. 이러한 experiment 는 중요하다. 널리 퍼진 견해에 따르면, vision Transformers 는 inductive bias 가 더 적기 때문에 더 큰 scale 로 pre-train 되었을 때 ConvNets 보다 더 잘 동작할 수 있기 때문이다.

  • 저자의 result 는 적절히 설계된 ConvNets 가 large dataset 으로 pre-train 되었을 때 vision Transformers 보다 열등하지 않음을 보여준다.
  • ConvNeXts 는 여전히 유사한 크기의 Swin Transformers 와 동등하거나 더 나은 성능을 보이며, throughput 은 약간 더 높다.
  • 추가로 ConvNeXt-XL model 은 87.8% 의 accuracy 를 달성하며, 이는 3842384^23842 에서의 ConvNeXt-L 보다 괜찮은 향상이다.
  • 이는 ConvNeXts 가 scalable architecture 임을 보여준다.
  • ImageNet-1K 에서 EfficientNetV2-L 은 Squeeze-and-Excitation 과 같은 advanced module 및 progressive training procedure 를 갖춘 searched architecture 로서 최고 성능을 달성한다.
  • 그러나 ImageNet-22K pre-training 을 사용하면 ConvNeXt 가 EfficientNetV2 를 능가할 수 있으며, 이는 large-scale training 의 중요성을 다시 보여준다.

Appendix B 에서 저자는 ConvNeXt 의 robustness 및 out-of-domain generalization result 를 논의한다.

3.3. Isotropic ConvNeXt vs. ViT​

이 ablation 에서 저자는 저자의 ConvNeXt block design 이 downsampling layer 가 없고 모든 depth 에서 동일한 feature resolution, 예를 들어 14×1414 \times 1414×14 를 유지하는 ViT-style isotropic architecture 에도 일반화될 수 있는지 살펴본다.

저자는 ViT-S/B/L 와 동일한 feature dimension 인 384/768/1024 를 사용하여 isotropic ConvNeXt-S/B/L 을 구성한다. depth 는 parameter 수와 FLOPs 를 맞추기 위해 18/18/36 으로 설정된다. block structure 는 동일하게 유지된다. Fig. 4 를 참조할 수 있다.

저자는 원래 ViTs 보다 향상된 training procedure 를 사용하는 DeiT 의 supervised training result 를 ViT-S/B 에 대해 사용하고, MAE 의 result 를 ViT-L 에 대해 사용한다. ConvNeXt model 은 이전과 동일한 setting 으로 학습되지만, warmup epoch 은 더 길다. 2242224^22242 resolution 에서의 ImageNet-1K result 는 Tab. 2 에 있다.

저자는 ConvNeXt 가 전반적으로 ViT 와 동등한 수준의 성능을 낼 수 있음을 관찰하며, 이는 저자의 ConvNeXt block design 이 non-hierarchical model 에 사용되더라도 경쟁력 있음을 보여준다.

4. Empirical Evaluation on Downstream Tasks

Object detection and segmentation on COCO.​

저자는 ConvNeXt backbone 을 사용하여 COCO dataset 에서 Mask R-CNN 과 Cascade Mask R-CNN 을 fine-tune 한다. Swin Transformer 를 따라 저자는 multi-scale training, AdamW optimizer, 3× schedule 을 사용한다. 더 자세한 내용과 hyper-parameter setting 은 Appendix A.3 에서 찾을 수 있다.

Tab. 3 은 Swin Transformer, ConvNeXt, 그리고 ResNeXt 와 같은 전통적인 ConvNet 을 비교한 object detection 및 instance segmentation result 를 보여준다.

  • 서로 다른 model complexity 전반에서 ConvNeXt 는 Swin Transformer 와 동등하거나 더 나은 성능을 달성한다.
  • 더 큰 model 인 ConvNeXt-B/L/XL 로 scale up 하고 ImageNet-22K 에서 pre-train 하면, 많은 경우 ConvNeXt 는 box 및 mask AP 측면에서 Swin Transformers 보다 유의하게 더 좋다. 예를 들어 +1.0+1.0+1.0 AP 이다.

Semantic segmentation on ADE20K.​

저자는 UperNet 과 함께 ADE20K semantic segmentation task 에서도 ConvNeXt backbone 을 평가한다. 모든 model variant 는 batch size 16 으로 160K iterations 동안 학습된다. 다른 experimental setting 은 따른다. 더 자세한 내용은 Appendix A.3 를 참조할 수 있다.

Tab. 4 에서 저자는 multi-scale testing 을 사용한 validation mIoU 를 보고한다.

  • ConvNeXt model 은 서로 다른 model capacity 전반에서 경쟁력 있는 성능을 달성할 수 있다.
  • 이는 저자의 architecture design 의 효과를 추가로 검증한다.

Remarks on model efficiency.​

유사한 FLOPs 조건에서 depthwise convolution 을 가진 model 은 dense convolution 만 가진 ConvNets 보다 더 느리고 더 많은 memory 를 소비하는 것으로 알려져 있다. 따라서 ConvNeXt 의 design 이 실제로 비효율적이 되는지 묻는 것은 자연스럽다. 논문 전체에서 보여주듯이, ConvNeXts 의 inference throughput 은 Swin Transformers 와 비슷하거나 이를 초과한다. 이는 classification 뿐 아니라 더 높은 resolution input 이 필요한 다른 task 에 대해서도 마찬가지이다. throughput/FPS 비교는 Tab. 1, Tab. 3 을 참조할 수 있다. 더 나아가 저자는 ConvNeXts 를 학습시키는 데 Swin Transformers 보다 더 적은 memory 가 필요하다는 점을 발견한다.

  • 예를 들어 ConvNeXt-B backbone 을 사용하여 Cascade Mask-RCNN 을 학습할 때, per-GPU batch size 2 조건에서 peak memory consumption 은 17.4 GB 이다.
  • 반면 Swin-B 의 reference number 는 18.5 GB 이다.

vanilla ViT 와 비교하면, ConvNeXt 와 Swin Transformer 모두 local computation 덕분에 더 유리한 accuracy-FLOPs trade-off 를 보인다. 주목할 점은, 이러한 향상된 efficiency 는 ConvNet inductive bias 의 결과이며, vision Transformers 의 self-attention mechanism 과 직접적으로 관련되어 있지는 않다는 것이다.

5. Related Work

Hybrid models.​

pre-ViT era 와 post-ViT era 모두에서, convolution 과 self-attention 을 결합한 hybrid model 은 활발히 연구되었다. ViT 이전에는 long-range dependency 를 포착하기 위해 ConvNet 에 self-attention/non-local module 을 추가하는 데 초점이 맞추어져 있었다. 원래의 ViT 는 hybrid configuration 을 처음으로 연구했으며, 이후의 많은 연구들은 explicit 하거나 implicit 한 방식으로 convolutional prior 를 ViT 에 다시 도입하는 데 초점을 맞추었다.

Recent convolution-based approaches.​

  • Han et al. 은 local Transformer attention 이 inhomogeneous dynamic depthwise conv 와 동등함을 보여준다.
  • 이후 Swin 의 MSA block 은 dynamic 또는 regular depthwise convolution 으로 대체되며, Swin 과 비슷한 성능을 달성한다.
  • 동시대 연구인 ConvMixer 는 small-scale setting 에서 depthwise convolution 이 유망한 mixing strategy 로 사용될 수 있음을 보여준다.
    • ConvMixer 는 최고의 result 를 얻기 위해 더 작은 patch size 를 사용하며, 이로 인해 throughput 은 다른 baseline 보다 훨씬 낮다.
  • GFNet 은 token mixing 을 위해 Fast Fourier Transform (FFT) 을 채택한다.
    • FFT 역시 global kernel size 와 circular padding 을 가진 convolution 의 한 형태이다.

많은 최근 Transformer 또는 ConvNet design 과 달리, 저자 연구의 주요 목표 중 하나는 standard ResNet 을 modernize 하여 state-of-the-art 성능을 달성하는 process 를 심층적으로 살펴보는 것이다.

6. Conclusions

2020 년대에 vision Transformers, 특히 Swin Transformers 와 같은 hierarchical model 은 generic vision backbone 으로서 선호되는 선택으로 ConvNets 를 앞서기 시작했다. 널리 퍼진 믿음은 vision Transformers 가 ConvNets 보다 더 정확하고, 더 efficient 하며, 더 scalable 하다는 것이다.

저자는 standard ConvNets 의 단순성과 efficiency 를 유지하면서도, 여러 computer vision benchmark 전반에서 state-of-the-art hierarchical vision Transformers 와 유리하게 경쟁할 수 있는 pure ConvNet model 인 ConvNeXts 를 제안한다.

어떤 면에서 저자의 관찰은 놀랍지만, ConvNeXt model 자체가 완전히 새로운 것은 아니다. 많은 design choice 는 지난 10 년 동안 각각 따로 검토되었지만, 집합적으로는 검토되지 않았다. 저자는 이 연구에서 보고한 새로운 result 가 널리 받아들여지는 몇 가지 견해에 도전하고, computer vision 에서 convolution 의 중요성을 다시 생각하게 만들기를 바란다.
